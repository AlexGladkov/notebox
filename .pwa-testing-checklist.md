# Чек-лист тестирования PWA

## Подготовка
- [ ] Собрать проект: `npm run build`
- [ ] Запустить preview: `npm run preview`
- [ ] Открыть в Chrome/Edge (лучшая поддержка PWA)

## Тестирование Service Worker

### 1. Регистрация SW
- [ ] Открыть DevTools → Application → Service Workers
- [ ] Проверить, что SW зарегистрирован и активен
- [ ] URL должен быть `/sw.js`

### 2. Кэширование статики
- [ ] В DevTools → Application → Cache Storage
- [ ] Проверить наличие кэша `workbox-precache-v2-*`
- [ ] Проверить, что закэшированы: HTML, CSS, JS, иконки

### 3. Офлайн-режим
- [ ] Загрузить приложение с интернетом
- [ ] В DevTools → Network выбрать "Offline"
- [ ] Перезагрузить страницу (Ctrl+R)
- [ ] Приложение должно загрузиться из кэша
- [ ] Проверить создание/редактирование заметок офлайн

## Тестирование Web App Manifest

### 1. Manifest доступен
- [ ] DevTools → Application → Manifest
- [ ] Проверить поля: name, short_name, icons, start_url
- [ ] Все иконки должны отображаться корректно

### 2. Установка PWA (Desktop Chrome/Edge)
- [ ] В адресной строке должна появиться иконка установки (⊕)
- [ ] Или появится промпт PWAInstallPrompt в правом нижнем углу
- [ ] Нажать "Установить"
- [ ] Приложение должно открыться в standalone-окне
- [ ] Проверить, что работает как десктопное приложение

### 3. Установка PWA (Android Chrome)
- [ ] Открыть на Android-устройстве
- [ ] Chrome покажет баннер "Добавить на главный экран"
- [ ] Или появится PWAInstallPrompt
- [ ] Установить приложение
- [ ] Проверить иконку на главном экране
- [ ] Открыть → должно работать в standalone режиме

### 4. iOS Safari (ограниченная поддержка)
- [ ] Открыть в Safari на iOS
- [ ] Нажать "Поделиться" → "На экран Домой"
- [ ] Проверить иконку и название
- [ ] Открыть → проверить работу

## Тестирование обновлений

### 1. Симуляция обновления
- [ ] Собрать проект первый раз
- [ ] Открыть в браузере, дождаться регистрации SW
- [ ] Изменить любой файл (например, заголовок в index.html)
- [ ] Пересобрать: `npm run build`
- [ ] Обновить страницу в браузере
- [ ] Должен появиться PWAUpdatePrompt вверху экрана
- [ ] Нажать "Обновить сейчас"
- [ ] Страница должна перезагрузиться с новым SW

## Тестирование Push-уведомлений

### 1. Функционал сохранен
- [ ] Service Worker должен содержать обработчики push/notificationclick
- [ ] Проверить в dist/sw.js наличие addEventListener('push')
- [ ] Навигация из уведомлений должна работать через postMessage

## Тестирование совместимости с окружением

### 1. Subpath deployment
- [ ] Установить `VITE_BASE_PATH=/test-app/`
- [ ] Пересобрать: `npm run build`
- [ ] Проверить, что все пути в manifest относительные
- [ ] Проверить, что SW работает с правильным scope

### 2. HTTP (без HTTPS)
⚠️ **Важно**: Service Worker требует HTTPS, кроме localhost
- [ ] На localhost (dev/preview) - SW работает
- [ ] На production HTTP - SW НЕ будет работать (ограничение браузеров)
- [ ] Для production нужен HTTPS

## Проверка производительности

- [ ] Lighthouse → PWA audit (должна быть оценка 100%)
- [ ] Проверить время загрузки из кэша (< 1 сек)
- [ ] Проверить размер precache (< 2 MB)

## Критические проблемы

Если что-то не работает:

1. **SW не регистрируется**
   - Проверить console на ошибки
   - Убедиться, что используется HTTPS или localhost
   - Проверить vite.config.ts настройки

2. **Кэш не работает**
   - Очистить все кэши в DevTools
   - Unregister старый SW
   - Пересобрать проект
   - Hard reload (Ctrl+Shift+R)

3. **Обновления не показываются**
   - Проверить registerType: 'prompt' в vite.config.ts
   - Проверить, что usePWA импортируется в App.vue

4. **Иконки не отображаются**
   - Проверить путь к иконкам в public/icons/
   - Проверить manifest.webmanifest
   - Пересобрать проект

## Совместимость браузеров

✅ **Полная поддержка:**
- Chrome 80+ (Desktop/Android)
- Edge 80+
- Samsung Internet 12+

⚠️ **Частичная поддержка:**
- Firefox 80+ (SW работает, установка ограничена)
- Safari 14+ (SW работает, установка через "На экран Домой")
- iOS Safari (ограничения на размер кэша, время жизни SW)

❌ **Не поддерживается:**
- IE 11
- Старые версии браузеров
- WebView в некоторых приложениях
