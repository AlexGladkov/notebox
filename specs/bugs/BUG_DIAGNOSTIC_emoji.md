# Диагностический отчет: Поиск emoji в пикере иконок не работает

**Дата:** 2026-04-01
**Severity:** MEDIUM
**Статус:** Диагностика завершена

---

## Краткое описание бага

При вводе текста в поле поиска emoji в пикере иконок результаты не отображаются. Пользователь вводит текст (например, "fire" или "smile"), но список emoji остается пустым.

---

## Корневая причина

**Файл:** `src/components/EmojiPicker.vue`
**Строки:** 99-100
**Уверенность:** HIGH

### Проблемный код

```typescript
const filteredEmojis = computed(() => {
  let emojis = emojisByCategory[selectedCategory.value] || []

  if (searchQuery.value) {
    const query = searchQuery.value.toLowerCase()
    emojis = Object.values(emojisByCategory).flat().filter(emoji => {
      return emoji.includes(query)  // <-- БАГ ЗДЕСЬ
    })
  }

  return emojis
})
```

### Почему это не работает

| Запрос пользователя | Значение emoji | Результат проверки | Причина |
|---------------------|----------------|-------------------|---------|
| "fire" | "🔥" | `false` | Строка "🔥" не содержит подстроку "fire" |
| "smile" | "😊" | `false` | Строка "😊" не содержит подстроку "smile" |

Код пытается найти текстовый запрос внутри Unicode-символа emoji, что невозможно. Emoji — это графические символы Unicode, а не текстовые строки.

## План исправления

### Шаг 1: Добавить маппинг emoji → ключевые слова

Создать структуру данных с ключевыми словами для поиска:

```typescript
const emojiKeywords: Record<string, string[]> = {
  '😀': ['smile', 'grin', 'happy', 'face', 'улыбка'],
  '😃': ['smile', 'happy', 'face', 'open', 'радость'],
  '🔥': ['fire', 'flame', 'hot', 'burn', 'огонь', 'пламя'],
  '😊': ['smile', 'blush', 'happy', 'улыбка'],
  // ... для всех emoji
}
```

### Шаг 2: Изменить логику фильтрации

```typescript
const filteredEmojis = computed(() => {
  let emojis = emojisByCategory[selectedCategory.value] || []

  if (searchQuery.value) {
    const query = searchQuery.value.toLowerCase()
    emojis = Object.values(emojisByCategory).flat().filter(emoji => {
      const keywords = emojiKeywords[emoji] || []
      return keywords.some(keyword => keyword.includes(query))
    })
  }

  return emojis
})
```

### Альтернативный подход: Использовать библиотеку emoji

Вместо ручного создания маппинга можно использовать npm-пакет с готовыми данными:
- `emojibase` — полная база данных emoji с ключевыми словами
- `emoji-js` — легковесная альтернатива

## Заключение

Баг вызван логической ошибкой в реализации поиска emoji. Код пытается найти текстовый запрос внутри Unicode-символов emoji, что невозможно. Для исправления необходимо добавить маппинг emoji → ключевые слова и изменить логику фильтрации для поиска по этим ключевым словам.

Исправление требует модификации одного файла и имеет низкий риск побочных эффектов.
