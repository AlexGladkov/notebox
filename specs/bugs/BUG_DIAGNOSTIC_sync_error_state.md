# Диагностический отчёт: Состояние ошибки синхронизации не сбрасывается

**Дата:** 2026-05-23  
**Серьёзность:** MEDIUM  
**Статус:** НЕ ВОСПРОИЗВОДИТСЯ (код содержит исправление)

---

## Bug Summary

Состояние `syncError` не сбрасывается после успешной синхронизации — пользователь видит красный индикатор ошибки даже когда синхронизация работает корректно.

---

## Root Cause

### Основная локация
**Файл:** `src/services/offline/syncService.ts`  
**Строка:** 35  
**Уверенность:** HIGH

### Анализ кода

Сброс состояния ошибки **РЕАЛИЗОВАН** в методе `processQueue()`:

```typescript
// syncService.ts:27-35
async processQueue(): Promise<void> {
  if (this.isProcessing.value) {
    console.log('Sync already in progress');
    return;
  }

  this.isProcessing.value = true;
  this.syncStatus.value = 'syncing';
  this.lastError.value = null;  // ← СБРОС ОШИБКИ
```

### Почему баг НЕ воспроизводится

1. **Сброс ошибки происходит** на строке 35 в `processQueue()`
2. Метод `sync()` в `useOffline.ts` вызывает `syncService.processQueue()`
3. При восстановлении сети watcher вызывает `sync()` → `processQueue()` → ошибка сбрасывается
4. Все пути выполнения проходят через `processQueue()`, где ошибка корректно обнуляется

### Потенциальная проблема (найдена, но не критична)

В методе `fullSync()` (строки 253-276) **НЕТ явного сброса** `lastError`:

```typescript
// syncService.ts:253-269
async fullSync(): Promise<void> {
  try {
    this.syncStatus.value = 'syncing';
    // ❌ НЕТ: this.lastError.value = null;
    
    await this.processQueue();  // ← Но вызывает processQueue, который сбрасывает
    // ...
  }
}
```

**Почему это не критично:** `fullSync()` внутри вызывает `processQueue()`, который сбрасывает ошибку.

## Fix Plan

### Текущий статус: Исправление не требуется

Баг не воспроизводится — код уже содержит корректную логику сброса состояния ошибки.

### Рекомендуемое улучшение (опционально)

**Шаг 1:** Добавить явный сброс `lastError` в начало `fullSync()` для консистентности

**Файл:** `src/services/offline/syncService.ts`  
**Строки:** 253-256

**Изменение:**
```typescript
async fullSync(): Promise<void> {
  try {
    this.syncStatus.value = 'syncing';
    this.lastError.value = null;  // ← ДОБАВИТЬ ЭТУ СТРОКУ
    
    await this.processQueue();
    // ...
```

**Обоснование:**
- Делает код более явным и самодокументируемым
- Защищает от будущих изменений в `processQueue()`
- Соответствует паттерну "defensive programming"

## Заключение
