# Диагностический отчёт: Pricing page shows 'undefined' for plan limits

**Дата диагностики:** 2026-05-24  
**Серьёзность:** HIGH  
**Статус:** НЕВОЗМОЖНО ИСПРАВИТЬ - ФУНКЦИОНАЛ ОТСУТСТВУЕТ

---

## Bug Summary

На странице /billing все тарифные планы (Starter, Pro, Business) якобы отображают 'undefined проектов' и 'undefined задач параллельно' вместо реальных значений лимитов.

---

## Root Cause

**ВЕРДИКТ: БАГ НЕВАЛИДНЫЙ - ФУНКЦИОНАЛ НЕ СУЩЕСТВУЕТ**

Страница `/billing` и весь функционал биллинга/тарифных планов **полностью отсутствует** в кодовой базе приложения.

**Ключевое наблюдение:**
- Маршрут `/billing` не зарегистрирован в роутере (`src/router/index.ts`)
- Нет Vue-компонентов для биллинга или тарифных планов
- Нет API endpoints для получения данных о планах
- Нет моделей данных с полями `maxProjects`, `maxParallelTasks`
- Нет таблиц в базе данных для подписок и планов
- Нет зависимостей платёжных систем (Stripe, Paddle и т.д.)

**Уровень уверенности:** 99% (HIGH)

---

## Consilium Findings

### Architect (Code Tracer)
- **Согласен с root cause:** Да
- **Резюме:** Комплексный анализ кода и git-истории (631 коммит) не выявил никаких следов реализации биллинга. Маршрут `/billing` никогда не существовал в роутере. Модель User не содержит полей для тарифных планов.

### Stack Expert
- **Согласен с root cause:** Да
- **Резюме:** Анализ зависимостей, конфигурации и окружения подтверждает отсутствие биллинг-функционала. Нет платёжных SDK, нет environment variables для биллинга, нет feature flags.

### Консенсус
**Полный консенсус:** Оба субагента независимо пришли к одинаковому выводу - функционал биллинга не реализован в приложении.

## Fix Plan

### Рекомендация: ЗАКРЫТЬ БАГ КАК НЕВАЛИДНЫЙ

Поскольку функционал биллинга не реализован, этот баг-репорт следует закрыть с одним из следующих статусов:

1. **"Invalid"** - если функционал не планировался
2. **"Not Applicable"** - если баг создан по ошибке
3. **"Won't Fix"** - если биллинг не входит в текущий scope проекта

### Если требуется реализация биллинга:

Создать отдельный Feature Request со следующим планом:

#### Step 1: Backend Database Schema
- Создать миграцию для таблиц `plans` и `user_subscriptions`
- Поля таблицы plans: id, name, maxProjects, maxParallelTasks, price, features

#### Step 2: Backend API
- Создать `BillingController` с endpoints:
  - `GET /api/billing/plans` - список доступных планов
  - `GET /api/billing/current` - текущий план пользователя
  - `POST /api/billing/subscribe` - подписка на план

#### Step 3: Frontend Components
- Создать `BillingView.vue` с отображением планов
- Создать компоненты карточек тарифов
- Интегрировать с роутером

#### Step 4: Payment Integration (опционально)
- Интегрировать Stripe или другую платёжную систему
- Добавить webhook-обработчики
