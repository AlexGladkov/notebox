| **Дата диагностики** | 2026-05-29 |

## Bug Summary

Компонент `QuickCaptureButton.vue` использует блокирующие вызовы `alert()` для показа уведомлений об успешном сохранении или ошибках. Это блокирует весь UI браузера до тех пор, пока пользователь не нажмёт OK, что критически ухудшает UX для функции Quick Capture, предназначенной для быстрой фиксации мыслей.

---

## Root Cause

### Первопричина
**Файл:** `src/components/QuickCapture/QuickCaptureButton.vue`  
**Строки:** 206-214

```typescript
function showSuccessNotification(message: string) {
  // TODO: Заменить на полноценную систему уведомлений
  alert(message);
}

function showErrorNotification(message: string) {
  // TODO: Заменить на полноценную систему уведомлений
  alert(message);
}
```

### Описание
Две вспомогательные функции используют синхронный, блокирующий API браузера `alert()` вместо существующей асинхронной системы toast-уведомлений. Комментарии TODO явно указывают на необходимость замены.

### Уверенность: ВЫСОКАЯ
- Баг подтверждён статическим анализом кода
- TODO-комментарии от разработчика подтверждают намерение заменить alert()
- В проекте уже существует готовая система toast-уведомлений


### Шаг 2: Заменить alert() на useToast() в QuickCaptureButton.vue

**Файл:** `src/components/QuickCapture/QuickCaptureButton.vue`

1. Добавить импорт useToast:
```typescript
import { useToast } from '../../composables/useToast';
```

2. Инициализировать composable:
```typescript
const { showSuccess, showError } = useToast();
```

3. Удалить функции showSuccessNotification и showErrorNotification (строки 206-214)

## Risk Assessment

### Низкий риск

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Стилистические конфликты z-index | Низкая | Низкое | ToastContainer использует z-index: 9999, выше чем модалы (z-50) |
| Дедупликация скрывает важные сообщения | Низкая | Низкое | 2-секундное окно дедупликации достаточно короткое |

### Минимальный риск

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|

### Что может сломаться

1. **Ничего критичного** - изменения локализованы
