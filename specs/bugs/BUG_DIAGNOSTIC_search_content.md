# Диагностический отчёт: Поиск не ищет по содержимому заметок

**Дата диагностики:** 2026-05-23  
**Severity:** MEDIUM  
**Статус:** Диагностика завершена, готово к исправлению

---

## Bug Summary

Поиск в приложении фильтрует заметки только по заголовку (`title`), полностью игнорируя содержимое заметок (`content`). Пользователь не может найти заметку по тексту внутри неё.

---

## Root Cause

**Файл:** `src/composables/useSearch.ts`  
**Строки:** 15-17

```typescript
return notes.value.filter(note =>
  note.title.toLowerCase().includes(query)
).sort((a, b) => b.updatedAt - a.updatedAt);
```

**Проблема:** Фильтр проверяет только `note.title.toLowerCase().includes(query)`, поле `note.content` не используется.

**Уверенность:** ВЫСОКАЯ — код явно показывает отсутствие поиска по content.

## Fix Plan

### Шаг 1: Модификация фильтра поиска
**Файл:** `src/composables/useSearch.ts`  
**Действие:** Изменить строки 15-17

**Текущий код:**
```typescript
return notes.value.filter(note =>
  note.title.toLowerCase().includes(query)
).sort((a, b) => b.updatedAt - a.updatedAt);
```

**Исправленный код:**
```typescript
return notes.value.filter(note =>
  note.title.toLowerCase().includes(query) ||
  note.content.toLowerCase().includes(query)
).sort((a, b) => b.updatedAt - a.updatedAt);
```

### Шаг 2 (опционально): Подсветка найденного фрагмента
