# Диагностический отчёт: Demo iframe shows 403 Forbidden instead of NoteBox app

**Дата:** 2026-05-24  
**Серьёзность:** HIGH  
**Статус:** КОРНЕВАЯ ПРИЧИНА ОПРЕДЕЛЕНА  
**Уверенность:** HIGH

---

## Краткое описание бага

Демо-превью iframe на странице проекта NoteBox показывает ошибку 403 Forbidden от nginx/1.29.5 вместо загрузки приложения NoteBox. URL `https://swarmhost.tech/notebox/` возвращает 403. Ошибка воспроизводится постоянно после загрузки страницы и после нажатия кнопки "Пересобрать".

---

## Корневая причина

### Основная проблема: Приложение не настроено для работы с path prefix `/notebox/`

Приложение собирается и конфигурируется для корневого пути `/`, но развёртывается на `https://swarmhost.tech/notebox/`. Это создаёт несколько каскадных сбоев:

| Файл | Строка | Проблема | Влияние |
|------|--------|----------|---------|
| `vite.config.ts` | 5 | `base: '/'` жёстко закодирован | Все ассеты ссылаются на абсолютные корневые пути |
| `src/router/index.ts` | 33 | `createWebHistory()` без параметра base | Vue Router не распознаёт маршруты на `/notebox/` |
| `src/App.vue` | 115 | `scope: '/'` в регистрации Service Worker | SW не регистрируется при работе с подпути |
| `.env.production` | 1 | `VITE_API_URL=` (пустой) | API-вызовы используют относительные пути |

### Почему появляется 403 на продакшене

Ошибка 403 Forbidden от nginx/1.29.5 на `swarmhost.tech` вызвана одной из причин:

1. **Внешний nginx настроен как файловый сервер** (не reverse proxy) — пытается обслуживать файлы из физической директории `/notebox/`, которой не существует
2. **Отключён directory listing** — nginx возвращает 403 вместо 404 при попытке доступа к директории без index-файла
3. **Активен security module** — многочисленные 404 запросы к несуществующим ассетам вызывают блокировку

**Важно:** Даже если внешний nginx настроить правильно и он вернёт 200 OK, Vue Router всё равно покажет 404, так как маршрут `/notebox/` не определён.

## Результаты Consilium-анализа

### Architect (Code Tracer)

**Согласие по корневой причине:** ДА

**Выводы:**
- Основная проблема в `vite.config.ts:5` — жёстко закодированный `base: '/'`
- Vue Router использует `createWebHistory()` без базового пути (строка 33)
- Service Worker регистрируется с `scope: '/'` (App.vue:115)
- Три разных паттерна обработки путей в кодовой базе создают несогласованность
- Недавние коммиты (a5a7be8, f361aea) частично исправили проблему, но не затронули корневую причину

### Stack Expert (Framework Analysis)

**Согласие по корневой причине:** ДА

**Выводы:**
- Vite `base` должен использовать переменную окружения, а не жёстко закодированное значение
- Vue Router должен использовать `import.meta.env.BASE_URL` в `createWebHistory()`
- Пустое значение `VITE_API_URL=` в `.env.production` усугубляет проблему
- Конфигурация nginx в репозитории корректна для SPA, но внешний nginx настроен неправильно
- Нет документации по развёртыванию с path prefix

### Консенсус

**Полное согласие** обоих субагентов по корневой причине:

1. Приложение собирается с `base: '/'`, но развёртывается на `/notebox/`
2. Vue Router не имеет базового пути, поэтому показывает 404 на `/notebox/`
3. Переменные окружения для subpath deployment не настроены
4. Внешний nginx на swarmhost.tech вероятно неправильно настроен

**Разногласий нет.**

## План исправления

### Шаг 1: Обновить vite.config.ts для динамического base path

**Файл:** `vite.config.ts`  
**Действие:** Изменить

**До:**
```typescript
export default defineConfig({
  base: '/',
  plugins: [vue()],
```

**После:**
```typescript
export default defineConfig({
  base: process.env.VITE_BASE_PATH || '/',
  plugins: [vue()],
```

### Шаг 2: Обновить Vue Router для использования BASE_URL

**Файл:** `src/router/index.ts`  
**Строка:** 33  
**Действие:** Изменить

**До:**
```typescript
const router = createRouter({
  history: createWebHistory(),
  routes
});
```

**После:**
```typescript
const router = createRouter({
  history: createWebHistory(import.meta.env.BASE_URL),
  routes
});
```

### Шаг 3: Обновить Service Worker registration

**Файл:** `src/App.vue`  
**Строка:** ~115  
**Действие:** Изменить

**До:**
```typescript
const registration = await navigator.serviceWorker.register('/sw.js', {
  scope: '/',
});
```

**После:**
```typescript
const registration = await navigator.serviceWorker.register(
  `${import.meta.env.BASE_URL}sw.js`,
  {
    scope: import.meta.env.BASE_URL,
  }
);
```

### Шаг 4: Добавить переменные окружения

**Файл:** `.env.example`  
**Действие:** Изменить

**Добавить:**
```
# Базовый путь для развёртывания (по умолчанию /)
# Для развёртывания на /notebox/ установить: /notebox/
VITE_BASE_PATH=/
```

### Шаг 5 (Опционально): Обновить nginx для явной поддержки subpath

**Файл:** `nginx/default.conf`  
**Действие:** Добавить комментарий/документацию

```nginx
# Для развёртывания с path prefix (например, /notebox/):
# 1. Собрать с VITE_BASE_PATH=/notebox/
# 2. Настроить внешний nginx как reverse proxy
```


   - Открыть `http://localhost/notebox/`
   - Убедиться, что показывается страница логина (не 404)

3. **Проверить маршрутизацию:**
   - Перейти на `http://localhost/notebox/login`
   - Перейти на `http://localhost/notebox/graph`
   - Проверить, что навигация работает

4. **Проверить iframe:**
   - Создать HTML с `<iframe src="http://localhost/notebox/">`
   - Убедиться, что приложение загружается в iframe

5. **Проверить DevTools:**
   - Network: Все запросы к ассетам должны использовать `/notebox/` префикс
   - Console: Не должно быть 404/403 ошибок

### Потенциальные риски

| Риск | Вероятность | Влияние | Митигация |

| Регрессия при развёртывании на корневом пути | Низкая | Высокое | По умолчанию `base: '/'` сохраняется |
| Неправильная конфигурация переменных окружения | Средняя | Высокое | Документация + валидация при сборке |

2. **Косвенное влияние:**
   - Все компоненты с `router-link`
   - API-клиент (если используется относительные пути)
   - Offline-синхронизация (зависит от Service Worker)

- Изменения минимальны и локализованы
- Обратная совместимость сохраняется (пустой `VITE_BASE_PATH` = `/`)
- Существующие тесты должны проходить без изменений

## Заключение

Баг вызван **отсутствием поддержки subpath deployment** в приложении. Приложение жёстко закодировано для работы с корневого пути `/`, но развёртывается на `https://swarmhost.tech/notebox/`.

Исправление требует:
1. Сделать Vite `base` path конфигурируемым через переменные окружения
2. Настроить Vue Router на использование `import.meta.env.BASE_URL`
3. Обновить Service Worker registration для динамического scope
4. Документировать процесс развёртывания с path prefix

**Ожидаемое время исправления:** 30-45 минут  
**Уровень риска:** Низкий  
**Требуется тестирование:** Среднее (проверка нескольких сценариев развёртывания)
