# Диагностический отчёт: Иконка заметки скрывается за backdrop

## Bug Summary

Иконка (emoji) заметки обрезается свойством `overflow: hidden` контейнера backdrop, вместо того чтобы отображаться на границе backdrop и контента с 50% выступом поверх обложки (как в Notion).

## Root Cause

**Файл:** `src/components/NoteCover.vue`
**Строка:** 198
**Уверенность:** ВЫСОКАЯ

```css
.cover-image {
  width: 100%;
  height: 200px;
  position: relative;
  border-radius: 8px 8px 0 0;
  overflow: hidden;  /* ← КОРНЕВАЯ ПРИЧИНА */
  background-position: center;
  background-size: cover;
}
```

### Механизм бага

1. `.cover-image` имеет `overflow: hidden` (строка 198 NoteCover.vue)
2. Иконка позиционируется внутри слота `#icon` с `bottom: -39px` (строки 166-172 NoteEditor.vue)
3. Иконка имеет размер 78px (строка 82 NoteIcon.vue)
4. `overflow: hidden` обрезает содержимое, выходящее за пределы контейнера высотой 200px
5. Результат: иконка не видна за пределами backdrop, хотя позиционирование корректное

## Fix Plan

### Шаг 1: Изменить overflow в NoteCover.vue (ОБЯЗАТЕЛЬНО)

**Файл:** `src/components/NoteCover.vue`
**Строка:** 198
**Действие:** Заменить `overflow: hidden` на `overflow: visible`

```css
/* БЫЛО: */
.cover-image {
  overflow: hidden;
}

/* СТАЛО: */
.cover-image {
  overflow: visible;
}
```

### Шаг 2: Проверить border-radius (ВОЗМОЖНО ПОТРЕБУЕТСЯ)

Если `overflow: visible` нарушит скругление углов backdrop-изображения, добавить вложенный элемент для фона:

```css
.cover-image::before {
  content: '';
  position: absolute;
  inset: 0;
  border-radius: inherit;
  overflow: hidden;
  background: inherit;
  z-index: -1;
}
```

### Шаг 3: Проверить позиционирование иконки

Убедиться, что `bottom: -39px` корректно работает с `overflow: visible`.
Текущее значение -39px = ~50% от 78px иконки, что даёт правильный эффект.

## Risk Assessment

### Низкий риск

- **Изменение:** Одна CSS-строка (overflow: hidden → overflow: visible)
- **Область влияния:** Только компонент NoteCover
- **Обратная совместимость:** Полная

### Потенциальные побочные эффекты

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Нарушение border-radius | Средняя | Использовать ::before псевдоэлемент |
| Выход других элементов за границы | Низкая | Элементы внутри не имеют отрицательных позиций |
| Визуальные артефакты | Низкая | Тестирование в обоих темах |

### Откат

В случае проблем — вернуть `overflow: hidden` и реализовать альтернативное решение (вынос иконки из слота).

Баг имеет простую корневую причину (CSS overflow:hidden) и требует минимального исправления. Оба субагента подтвердили диагноз с высокой уверенностью. Исправление низкорискованное и локализованное в одном компоненте.
