## Bug Summary

Баг описывает ситуацию, когда кнопка "Open Project Manager" создаёт полноэкранный overlay с классами `bg-black/30 z-40`, который перехватывает все pointer-события и блокирует взаимодействие со страницей. Escape не закрывает overlay, требуется перезагрузка страницы.

---

## Consilium Findings

### Architect (Code Tracer)

**Ключевые находки:**
- "Project Manager" компонент не существует
- Проанализировано 10 overlay-компонентов в кодовой базе
- Найдены **потенциальные проблемы** в других модальных компонентах:
  - `ConfirmDialog.vue` - отсутствует `@click.self` и Escape handler
  - `CsvImportDialog.vue` - аналогичные проблемы
- QuickCapture модали реализованы корректно (TextCaptureModal, PhotoCaptureModal и др.)

**Вывод:** Баг как описан не существует, но есть связанные UX-проблемы в других модалях.

### Stack Expert

**Ключевые находки:**
- Стек: Vue 3.4.0 + Vite 5.0.0 + Tailwind CSS 3.4.0 + Pinia 2.3.1
- Tailwind config не содержит кастомных z-index расширений
- Текущий код использует `bg-opacity-50` и `z-50`, а не `bg-black/30 z-40`
- Коммит `a236081` уже исправил похожий баг с z-index в Database компонентах
- Архитектура модалей использует Vue 3 Teleport и Transition корректно

**Вывод:** Технический стек в порядке, описанные CSS-классы не соответствуют реальному коду.
