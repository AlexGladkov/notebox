# Диагностический отчёт: API client не обрабатывает null/undefined данные

**Дата:** 2026-05-23  
**Severity:** HIGH (заявленная)  
**Статус:** FALSE POSITIVE — баг не подтверждён

---

## Bug Summary

Описание бага утверждает, что в `composables/useApi.ts` функции get/post/put/delete не проверяют `response.data` на null/undefined перед возвратом. **Анализ показал, что файл не существует, а реальный API клиент в `src/api/client.ts` уже имеет полную защиту от null/undefined.**

---

## Root Cause

### Статус: БАГ НЕ СУЩЕСТВУЕТ

**Указанный файл:** `composables/useApi.ts` — **НЕ НАЙДЕН** в кодовой базе

**Реальный API клиент:** `src/api/client.ts:49-51`

```typescript
if (data.data === null || data.data === undefined) {
  throw new ApiError('NO_DATA', 'No data returned from server');
}
```

Защита реализована корректно и существует с момента создания файла (коммит `1ed0631` от 2026-02-11).

## Consilium Findings

### Architect (Code Tracer)

**Согласие:** ✅ Баг не подтверждён

**Ключевые находки:**
- Файл `composables/useApi.ts` не существует в проекте
- Реальный API клиент: `src/api/client.ts`
- Защита от null/undefined реализована в функции `handleResponse<T>()` (строки 49-51)
- Все 10 API модулей (`notes.ts`, `databases.ts`, `files.ts` и др.) используют защищённый `apiClient`
- Git история показывает, что защита существует с первого коммита

**Data Flow:**
1. `apiClient.get/post/put/patch/delete()` → вызов `handleResponse<T>()`
2. `handleResponse()` проверяет `response.ok`, парсит JSON
3. Проверка `data.data === null || data.data === undefined`
4. При null/undefined бросается `ApiError('NO_DATA', ...)`
5. Возврат типизированных данных `data.data`

### Stack Expert

**Согласие:** ✅ Баг не подтверждён

- Клиент правильно это обрабатывает

## Дополнительные находки

### Проблема с демо-логином (не связана с багом)

При тестировании обнаружена ошибка 403 при попытке демо-логина. Это проблема конфигурации тестового окружения, не связанная с обработкой null/undefined.

### Архитектура API клиента

Проект использует централизованный паттерн API клиента:
- Единая точка входа: `src/api/client.ts`
- Все HTTP методы проходят через `handleResponse()`
- Типизированная обработка ошибок через `ApiError`
- Все модули (`notes.ts`, `databases.ts` и др.) используют `apiClient`
