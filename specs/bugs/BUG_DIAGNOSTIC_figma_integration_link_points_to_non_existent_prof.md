# Диагностический отчёт: Figma integration link points to non-existent /profile?tab=integrations page

**Серьёзность:** MEDIUM  
**Дата диагностики:** 2026-05-24  
**Статус:** Диагностирован

---

## Bug Summary

Ссылка на интеграцию Figma указывает на несуществующий маршрут `/profile?tab=integrations`, который возвращает ошибку 404. Однако сама ссылка "Подключите Figma в настройках" **не найдена в текущей кодовой базе**, что указывает на устаревшее описание бага или удалённый функционал.

---

## Root Cause

**Основная причина:** Отсутствие маршрута `/profile` в конфигурации роутера

**Файл:** `src/router/index.ts` (строки 5-30)

```typescript
const routes: RouteRecordRaw[] = [
  { path: '/login', ... },           // LoginView
  { path: '/', ... },                // MainView (home)
  { path: '/graph', ... },           // GraphView
  { path: '/:pathMatch(.*)*', ... }  // NotFoundView (catch-all)
];
// ❌ /profile route never defined
```

**Вторичная причина:** Отсутствие секции "Интеграции" в настройках

**Файл:** `src/components/settings/SettingsSidebar.vue` (строки 93-97)

```typescript
const sections: SettingsSection[] = [
  { id: 'profile', label: 'Профиль', icon: UserIcon },
  { id: 'appearance', label: 'Внешний вид', icon: PaletteIcon },
  { id: 'notifications', label: 'Уведомления', icon: BellIcon },
  // ❌ Нет секции 'integrations'
];
```

**Уровень уверенности:** HIGH

## Consilium Findings

### Architect (Code Tracer)

**Согласие с корневой причиной:** ДА

**Ключевые находки:**
- Маршрут `/profile` полностью отсутствует в `src/router/index.ts`
- Настройки профиля реализованы через модальное окно (SettingsModal), а не через отдельный маршрут
- Ссылка "Подключите Figma в настройках" **не найдена** ни в одном файле кодовой базы
- Коммит f361aea (2026-05-23) добавил catch-all обработчик 404, поэтому `/profile` теперь корректно возвращает страницу ошибки
- Модальная архитектура настроек не поддерживает query-параметры типа `?tab=integrations`

**Рекомендация:** Добавить секцию интеграций в модальное окно настроек (Option B) или создать полноценный маршрут `/profile` (Option A)

### Stack Expert (Vue.js/Vite)

**Согласие с корневой причиной:** ДА

**Ключевые находки:**
- Проект использует Vue 3.4.0 + Vue Router 4.2.0 + Vite 5.0.0
- Роутер настроен в режиме history с корректным catch-all маршрутом
- Nginx SPA fallback настроен правильно (`try_files $uri $uri/ /index.html`)
- Поиск по "Figma", "integrations", "/profile" не дал результатов в компонентах
- Переменные окружения не содержат флагов для интеграций
- Компонент IntegrationsSection.vue не существует

**Рекомендация:** Либо удалить несуществующие ссылки, либо реализовать функционал интеграций

## Conclusion

Баг с несуществующим маршрутом `/profile?tab=integrations` **подтверждён**, но его источник (ссылка "Подключите Figma в настройках") **не найден** в текущей кодовой базе. Это указывает на один из сценариев:

1. Ссылка была удалена в предыдущих коммитах
2. Описание бага устарело
3. Ссылка генерируется динамически в условиях, не воспроизведённых при тестировании

**Рекомендация:** Применить вариант A (добавить редирект) для предотвращения 404, и уточнить с product owner необходимость полной реализации интеграций (вариант B).
