# Диагностический отчёт: Белый экран на маршруте /notes

**Дата:** 2026-05-23  
**ID бага:** white-screen-on-notes-route-with-valid-session  
**Серьёзность:** ВЫСОКАЯ  
**Статус:** Корневая причина идентифицирована  

---

## Краткое описание бага

При прямом переходе на URL `/notes` приложение показывает полностью белый экран без контента, ошибок или индикаторов загрузки. DOM полностью пуст.

---

## Корневая причина

**Файл:** `src/router/index.ts`  
**Строки:** 5-24  
**Уверенность:** ВЫСОКАЯ  

### Суть проблемы

**Маршрут `/notes` НЕ СУЩЕСТВУЕТ** в конфигурации Vue Router.

Текущие маршруты:
```typescript
const routes: RouteRecordRaw[] = [
  { path: '/login', ... },    // Страница логина
  { path: '/', ... },         // Главная страница
  { path: '/graph', ... }     // Граф заметок
  // ❌ НЕТ маршрута /notes
  // ❌ НЕТ catch-all маршрута
];
```

### Механизм бага

1. Пользователь переходит на `/notes`
2. Vue Router ищет маршрут → не находит
3. `to.matched` = пустой массив `[]`
4. Auth guard: `requiresAuth = [].some(...)` = `false`
5. Guard возвращает `undefined` (разрешает навигацию)
6. Vue Router продолжает, но компонента для рендеринга нет
7. `<router-view>` рендерит пустоту
8. **Результат: белый экран**

## План исправления

### Шаг 1: Добавить catch-all маршрут (КРИТИЧНО)

**Файл:** `src/router/index.ts`

```typescript
const routes: RouteRecordRaw[] = [
  // ... существующие маршруты ...
  {
    path: '/:pathMatch(.*)*',
    name: 'NotFound',
    component: () => import('../views/NotFoundView.vue'),
    meta: { requiresAuth: false }
  }
];
```

### Шаг 2: Создать NotFoundView компонент (КРИТИЧНО)

**Файл:** `src/views/NotFoundView.vue`

```vue
<template>

    <h1 class="text-4xl font-bold mb-4">404</h1>
    <p class="text-gray-600 mb-6">Страница не найдена</p>
    <router-link to="/" class="text-blue-500 hover:underline">
      Вернуться на главную
    </router-link>
  </div>
</template>
```

### Шаг 3: Улучшить auth guard (ВЫСОКИЙ ПРИОРИТЕТ)

**Файл:** `src/router/guards.ts`

```typescript
export async function authGuard(to: RouteLocationNormalized) {
  const authStore = useAuthStore();

  if (!authStore.isInitialized) {
    await authStore.checkAuth();
  }

  // НОВОЕ: Явная обработка неопределённых маршрутов
  if (to.matched.length === 0) {
    return { path: '/', replace: true };
  }

}
```

## Заключение

Это **НЕ race condition** и **НЕ проблема авторизации**. Это отсутствие маршрута `/notes` в комбинации с поведением Vue Router по умолчанию — молчаливый рендеринг пустого `<router-view>` когда маршрут не найден.

Исправление простое и низкорискованное: добавить catch-all маршрут с компонентом NotFoundView.
