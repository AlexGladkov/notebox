# Диагностический отчёт: Очистка уведомления экспорта падает если parent node уже удалён

**Дата анализа**: 2026-05-23  
**Серьёзность**: LOW  
**Статус воспроизведения**: ❌ НЕ ВОСПРОИЗВЕДЁН (баг уже исправлен)

---

## Bug Summary

Функция `showNotification()` в компоненте `ExportButton.vue` использовала `setTimeout()` для автоматического удаления DOM-элемента уведомления без проверки наличия элемента в DOM. Если пользователь закрывал уведомление вручную до срабатывания таймера, возникала ошибка `DOMException: Failed to execute 'removeChild' on 'Node': The node to be removed is not a child of this node`.

---

## Root Cause

**Файл**: `src/components/ExportButton.vue`  
**Функция**: `showNotification()` (строки 121-179)  
**Проблемная строка**: 175 (оригинально без проверки parentNode)  
**Уверенность**: HIGH

### Механизм Race Condition

```
Сценарий ошибки:
┌─────────────────────────────────────────────────────────────────┐
│ T=0ms        │ T=500ms              │ T=3300ms                  │
│ Создание     │ Пользователь         │ setTimeout срабатывает    │
│ notification │ закрывает вручную    │ removeChild() → ❌ ERROR  │
│              │ → parentNode = null  │                           │
└─────────────────────────────────────────────────────────────────┘
```

### Оригинальный код (с багом)

```javascript
// Коммит 0d13d22
setTimeout(() => {
  notification.style.animation = 'slideOut 0.3s ease';
  setTimeout(() => {
    document.body.removeChild(notification);  // ❌ БАГ: нет проверки
  }, 300);
}, 3000);
```

### Исправленный код (текущая версия)

```javascript
// Коммит 0ca37eb / fe5b97d
setTimeout(() => {
  notification.style.animation = 'slideOut 0.3s ease';
  setTimeout(() => {
    if (notification.parentNode) {  // ✅ ИСПРАВЛЕНО
      document.body.removeChild(notification);
    }
  }, 300);
}, 3000);
```
