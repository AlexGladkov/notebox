# Диагностический отчёт: /profile рендерит пустой DOM и перенаправляет на /projects

**Дата анализа:** 2026-05-24  
**Серьёзность:** MEDIUM  
**Уровень уверенности:** HIGH  

---

## Bug Summary

Маршрут `/profile` не существует в конфигурации роутера — при переходе отображается страница 404, а не настройки профиля. Функциональность профиля реализована через модальное окно, доступное по клику на имя пользователя.

---

## Root Cause

**Файл:** `src/router/index.ts:5-30`  
**Описание:** Маршрут `/profile` никогда не был реализован. В роутере определены только 4 маршрута: `/login`, `/`, `/graph` и catch-all `/:pathMatch(.*)* ` для 404.

**Ключевой код (src/router/index.ts):**
```typescript
const routes: RouteRecordRaw[] = [
  { path: '/login', ... },           // LoginView
  { path: '/', ... },                // MainView (home)
  { path: '/graph', ... },           // GraphView
  { path: '/:pathMatch(.*)*', ... }  // NotFoundView (catch-all)
];
// ❌ Маршрут /profile ОТСУТСТВУЕТ
```

**Архитектурное решение:** Профиль реализован как модальное окно (`SettingsModal.vue`), вызываемое из `MainView.vue` при клике на компонент `UserProfile`.

## Risk Assessment

### Низкий риск:
- Добавление нового маршрута не влияет на существующую функциональность
- ProfileSection уже протестирован в составе модального окна
- Auth guard уже настроен для защиты маршрутов

### Потенциальные проблемы:
1. **Дублирование:** Профиль будет доступен и через модальное окно, и через маршрут — нужно решить, оставлять ли оба варианта
2. **Навигация:** Может потребоваться добавить ссылку на `/profile` в навигацию
3. **Состояние:** Убедиться, что состояние синхронизируется между модальным окном и страницей профиля

### Митигация:
- Переиспользовать существующий ProfileSection
- Использовать единый authStore для управления состоянием
- Протестировать синхронизацию между модальным окном и страницей

## Дополнительные заметки

### Архитектурный контекст:
Текущая реализация через модальное окно (паттерн Notion) — это валидный и широко используемый подход. Он обеспечивает:
- Быстрый доступ без потери контекста
- Меньше переходов между страницами
- Консистентный UX в рамках одного окна

### Рекомендация:
Перед реализацией fix'а рекомендуется уточнить у автора бага или product owner'а:
1. Действительно ли требуется отдельный маршрут `/profile`?
2. Достаточно ли текущей реализации через модальное окно?
3. Если нужен маршрут, должен ли он заменить модальное окно или дополнить его?
