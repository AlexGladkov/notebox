# Спецификация: Экспорт заметок в PDF

## Идентификатор
F-266b41ca-add-export-notes-to-pdf

## Статус
Готово к реализации

## Резюме
Добавить функциональность экспорта заметок в PDF-файл. Пользователь сможет скачать текущую открытую заметку в формате PDF с сохранением форматирования, заголовка, тегов и визуального оформления.

## Контекст
- **Проект**: Notebox — приложение для заметок
- **Стек Frontend**: Vue 3, Vite, TailwindCSS, TipTap (ProseMirror)
- **Стек Backend**: Kotlin, Spring Boot
- **Формат контента**: TipTap JSON (заголовки H1-H3, списки, чек-боксы, цитаты, блоки кода, callouts, ссылки, подсветка)

## Требования

### Функциональные требования

#### FR-1: Кнопка экспорта в PDF
- Добавить кнопку "Экспорт в PDF" в панель инструментов редактора заметки
- Кнопка доступна только при открытой заметке
- Иконка: документ с стрелкой вниз или аналогичная

#### FR-2: Генерация PDF
- PDF генерируется на стороне клиента (без обращения к серверу)
- Использовать библиотеку html2pdf.js для конвертации HTML в PDF
- Имя файла: `{название_заметки}.pdf` (транслитерация кириллицы в латиницу)

#### FR-3: Содержимое PDF
PDF должен включать:
- Заголовок заметки (крупным шрифтом)
- Теги заметки (если есть)
- Дату последнего изменения
- Основной контент с сохранением форматирования:
  - Заголовки H1, H2, H3
  - Параграфы
  - Маркированные и нумерованные списки
  - Списки задач (чек-боксы) — отмечать как [x] или [ ]
  - Цитаты (blockquote)
  - Блоки кода с моноширинным шрифтом
  - Callouts (info/warning/error/success) с соответствующими цветами
  - Ссылки (отображать как текст, URL в скобках)
  - Подсветка текста

#### FR-4: Стили PDF
- Шрифт: системный sans-serif для основного текста, monospace для кода
- Размер страницы: A4
- Поля: 20mm со всех сторон
- Цветовая схема: светлая (независимо от темы приложения)

### Нефункциональные требования

#### NFR-1: Производительность
- Генерация PDF для заметки до 10 страниц должна занимать менее 3 секунд
- Показывать индикатор загрузки во время генерации

#### NFR-2: Совместимость
- Работа в Chrome, Firefox, Safari, Edge (последние 2 версии)
- Корректная обработка кириллицы и emoji

#### NFR-3: Доступность
- Кнопка экспорта должна иметь aria-label
- Уведомление об успешном скачивании

## Область применения (Scope)

### В scope (MVP)
- Экспорт одной текущей заметки
- Базовое форматирование TipTap-контента
- Генерация на клиенте
- Автоматическое скачивание файла

### Вне scope (будущие улучшения)
- Экспорт нескольких заметок в один PDF
- Экспорт заметки с дочерними страницами (иерархия)
- Включение обложки (backdrop) заметки в PDF
- Включение встроенных баз данных (DatabaseBlock)
- Настройки экспорта (размер страницы, поля, шрифт)
- Генерация PDF на сервере
- Экспорт в другие форматы (Word, Markdown)

### Структура HTML для PDF
```html
<div class="pdf-export">
  <h1 class="pdf-title">{title}</h1>
  <div class="pdf-meta">

  </div>
  <div class="pdf-content">

  </div>

```

## Пограничные случаи и риски

### Edge Cases
1. **Пустая заметка** — экспортировать PDF только с заголовком и датой
2. **Очень длинный заголовок** — обрезать до 100 символов в имени файла
3. **Специальные символы в заголовке** — удалять символы недопустимые в именах файлов (`/\:*?"<>|`)
4. **Большая заметка** — может потребоваться больше времени, показывать прогресс
5. **Заметка без тегов** — не показывать секцию тегов

### Риски
1. **html2pdf.js может некорректно рендерить сложные элементы**
   - Митигация: тестирование всех типов блоков TipTap
   
2. **Проблемы с кириллицей в имени файла**
   - Митигация: транслитерация имени файла

3. **Встроенные базы данных не будут экспортированы**
   - Митигация: показывать placeholder "[База данных]" с именем
