# Спецификация: Улучшение empty states с иллюстрациями и CTA

## Идентификатор
F-2c861fbf

## Сводка
Переработка пустых состояний во всех ключевых разделах приложения NoteBox для улучшения UX. Каждое пустое состояние будет содержать информативную иллюстрацию, понятный CTA (Call-to-Action) и текст, объясняющий ценность раздела. Дополнительно реализуется confirmation modal при удалении заметок с дочерними страницами.

## Собранные требования

### Анализ текущего состояния

**Существующий компонент EmptyState** (`src/components/EmptyState.vue`):
- Простой компонент с единственной SVG иконкой документа
- Принимает только `message: string`
- Не поддерживает CTA, пользовательские иллюстрации, описания

**MainView** (`src/views/MainView.vue`):
- Пустое состояние списка страниц: простой текст "Создайте первую страницу" (строка 72-74)
- Пустой результат поиска: "Ничего не найдено" (строки 41-43)
- Нет иллюстраций и CTA

**GraphView** (`src/views/GraphView.vue`):
- Есть пустое состояние с SVG иконкой и двумя строками текста (строки 32-56)
- Нет CTA для создания страниц
- Нет объяснения как создаются связи

**DatabaseTable** (`src/components/BlockEditor/DatabaseTable.vue`):
- Пустое состояние: "Нет записей" / "Нет записей, соответствующих фильтру" (строки 63-68, 115-119)
- Нет иллюстрации, нет шаблонов, нет подсказок

**ConfirmDialog** (`src/components/ConfirmDialog.vue`):
- Базовый dialog с title и message
- Не показывает информацию о дочерних страницах

### Визуальный стиль (решение)

Использовать **минималистичные SVG иллюстрации** в едином стиле:
- Контурные линии (stroke-based)
- Палитра: серый для контуров, акцентный синий для CTA элементов
- Размер иллюстраций: 120-160px
- Поддержка тёмной темы через CSS переменные и Tailwind классы

## Детальный план реализации

### 1. Переработка компонента EmptyState

```typescript
// Новые props для EmptyState.vue
interface Props {
  title: string;                          // Заголовок (обязательный)
  description?: string;                   // Описательный текст
  icon?: Component;                       // Пользовательская иконка/иллюстрация (Vue компонент)
  ctaLabel?: string;                      // Текст кнопки CTA
  ctaVariant?: 'primary' | 'secondary';   // Стиль кнопки
  hints?: string[];                       // Список подсказок (для поиска)
}

// Emits
interface Emits {
  (e: 'cta-click'): void;
}
```

**Структура шаблона:**
```html
<div class="empty-state">
  <component :is="icon" v-if="icon" class="illustration" />
  <DefaultIcon v-else class="illustration" />
  <h3 class="title">{{ title }}</h3>
  <p v-if="description" class="description">{{ description }}</p>
  <ul v-if="hints?.length" class="hints">
    <li v-for="hint in hints">{{ hint }}</li>
  </ul>
  <button v-if="ctaLabel" @click="emit('cta-click')" class="cta-button">
    {{ ctaLabel }}
  </button>
</div>
```

### 5. Confirmation Modal для удаления с дочерними страницами

**Изменения в ConfirmDialog:**
```typescript
// Новые props
interface Props {
  show: boolean;
  title: string;
  message: string;
  details?: {                    // Новое
    childrenCount?: number;      // Количество дочерних страниц
    itemsList?: string[];        // Список названий (до 5 элементов)
  };
  confirmLabel?: string;         // Текст кнопки подтверждения (по умолчанию "Удалить")
  confirmVariant?: 'danger' | 'warning';  // Стиль кнопки
}
```

**Шаблон деталей:**
```html
<div v-if="details?.childrenCount" class="details-section">
  <p class="warning-text">
    Также будут удалены {{ details.childrenCount }} дочерних страниц:
  </p>
  <ul v-if="details.itemsList" class="items-list">
    <li v-for="item in details.itemsList.slice(0, 5)">{{ item }}</li>
    <li v-if="details.itemsList.length > 5">
      ...и ещё {{ details.itemsList.length - 5 }}
    </li>
  </ul>
</div>
```

**Интеграция в MainView/NoteTree:**
При удалении заметки с дочерними страницами:
1. Вызвать `getChildrenCount(noteId)` для получения количества
2. Вызвать `getAllDescendants(noteId)` для списка названий
3. Передать данные в ConfirmDialog через props

## Критерии приёмки

1. **Иллюстрации**
   - [ ] Каждый empty state содержит уникальную SVG иллюстрацию
   - [ ] Иллюстрации адаптированы под тёмную тему
   - [ ] Размер иллюстраций единообразный (120-160px)

2. **CTA кнопки**
   - [ ] Главная страница (пустой список): кнопка "Создать заметку" работает
   - [ ] Граф связей: кнопка "Создать страницу" переводит на главную и создаёт заметку
   - [ ] База данных: кнопка "Добавить запись" добавляет строку

3. **Текстовый контент**
   - [ ] Каждый empty state имеет заголовок и описание
   - [ ] Описания объясняют ценность функции, а не просто её пустоту
   - [ ] Пустой поиск показывает подсказки по синтаксису

4. **Confirmation Modal**
   - [ ] При удалении заметки с дочерними показывается их количество
   - [ ] Показывается список названий дочерних страниц (до 5)
   - [ ] Если больше 5, показывается "...и ещё N"

5. **Общее**
   - [ ] Все тексты на русском языке
   - [ ] Поддержка светлой и тёмной темы
   - [ ] Корректная работа на мобильных устройствах (responsive)

## Edge Cases и риски

### Edge Cases

1. **Очень длинные названия заметок в Confirmation Modal**
   - Обрезать с "..." после 50 символов

2. **Большое количество дочерних страниц (100+)**
   - Показывать только количество, не загружать все названия
   - Лимит списка: 5 названий + "...и ещё N"

3. **Пустой поиск по одному символу**
   - Показывать стандартный empty state без специальных подсказок

4. **Граф с одной заметкой без связей**
   - Показывать заметку как одиночный узел (это не empty state)

### Риски

1. **Размер SVG иллюстраций**
   - Риск: увеличение размера бандла
   - Митигация: использовать inline SVG, оптимизировать через SVGO

2. **Консистентность стиля**
   - Риск: разнородные иллюстрации ухудшают UX
   - Митигация: создать все иллюстрации в едином стиле, документировать guidelines

## Вне скоупа (Out of Scope)

- Анимации появления empty states
- A/B тестирование текстов
- Локализация на другие языки (кроме русского)
- Интерактивные туториалы/онбординг
- Кастомизация иллюстраций пользователем
- Empty state для корзины/удалённых заметок (если такая функция появится)
