# F-2f33261c: Шаринг страниц по ссылке (только чтение)

## Краткое описание

Добавление возможности поделиться страницей (заметкой) по публичной ссылке. Посетители смогут просматривать страницу без авторизации в режиме только чтение.

## Требования (Acceptance Criteria)

1. **Кнопка "Поделиться"** в меню страницы (рядом с кнопкой экспорта)
2. **Генерация уникальной публичной ссылки** при включении шаринга
3. **Страница доступна без авторизации** в режиме только чтение
4. **Возможность отключить публичный доступ** (отозвать ссылку)
5. **Индикатор что страница опубликована** (визуальная метка)

## Архитектурные решения

### 1. Формат публичной ссылки

**Выбор:** UUID-токен  
**Формат:** `/share/{shareToken}`  
**Пример:** `/share/a1b2c3d4-e5f6-7890-abcd-ef1234567890`

**Обоснование:**
- Уникальный токен не связан с ID страницы (безопасность)
- Невозможно угадать ссылку перебором
- Можно перегенерировать токен при необходимости

### 2. Расширение модели Note

Добавить опциональное поле `shareToken`:

```typescript
interface Note {
  // ... существующие поля
  shareToken?: string | null;  // UUID токен для публичного доступа
}
```

### 3. API эндпоинты (требуют реализации на бэкенде)

```
POST   /api/notes/{id}/share     - Включить шаринг, вернуть shareToken
DELETE /api/notes/{id}/share     - Отключить шаринг
GET    /api/public/notes/{token} - Получить публичную страницу (без авторизации)
```

### 4. Маршрутизация

Новый публичный роут без авторизации:

```typescript
{
  path: '/share/:token',
  name: 'shared-note',
  component: () => import('../views/SharedNoteView.vue'),
  meta: { requiresAuth: false }
}
```

## Особенности публичного просмотра

### Что показывается:
- Заголовок страницы
- Иконка страницы (если есть)
- Обложка (backdrop) если есть

### Что НЕ показывается / заблокировано:

- Вложенные страницы (только текущая)
- Навигация по приложению
- Действия (экспорт, напоминания, избранное)
- Wiki-ссылки (показываются как текст, не кликабельны)
- Backlinks

## Edge Cases и обработка ошибок

### Edge Cases:
1. **Удаление страницы** — публичная ссылка перестаёт работать (404)
2. **Отключение шаринга** — ссылка сразу перестаёт работать
3. **Одновременное редактирование** — публичная версия показывает данные на момент запроса
4. **Пустая страница** — показать placeholder "Страница пуста"
5. **Очень длинный контент** — стандартная прокрутка
6. **Офлайн режим** — показать сообщение об отсутствии подключения

### Ошибки:
1. **Недействительный токен** — 404 с понятным сообщением
2. **Ошибка сети** — retry логика + сообщение
3. **Ошибка при копировании** — fallback на ручное выделение

## Ограничения среды развертывания

Приложение разворачивается:
- На subpath `/<project-slug>/`
- За nginx reverse proxy по HTTP (не HTTPS)

### Учтено в реализации:
- Публичная ссылка строится относительно `window.location.origin`
- Роутер уже использует `createWebHistory()` без base
- Для subpath: использовать `import.meta.env.BASE_URL`

5. Защита паролем

7. QR-код для ссылки
8. Права доступа (только чтение vs редактирование)
