# Спецификация: Улучшения слэш-команд

**ID задачи:** F-1a69c41d
**Дата:** 2026-02-24
**Статус:** Готово к реализации

## Краткое описание

Две связанные задачи по улучшению системы слэш-команд:
1. Исправление бага: клавиша Enter не выбирает элемент в выпадающем меню слэш-команд
2. Новая функциональность: команда `/nested` для создания вложенных (дочерних) заметок

---

## Требования (результаты интервью)

### 1. Исправление Enter в слэш-меню

**Проблема:**
При открытом меню слэш-команд нажатие Enter не выбирает подсвеченный пункт. Вместо этого Enter либо создаёт новую строку, либо ничего не делает. Пользователь вынужден кликать мышью для выбора команды.

**Причина:**
TipTap/ProseMirror перехватывает события клавиатуры на уровне редактора раньше, чем document-level listener в `SlashCommandMenu.vue`.

**Решение:**
Добавить обработку клавиатуры непосредственно в расширение `SlashCommand.ts` через `addKeyboardShortcuts()` или через props плагина ProseMirror. Когда меню активно, Enter/ArrowUp/ArrowDown/Escape должны обрабатываться расширением, а не редактором.

**Браузер для тестирования:** Chrome (воспроизводится стабильно)

### 2. Команда /nested для вложенных заметок

**Пользовательский сценарий:**
1. Пользователь вводит `/nested` в редакторе
2. Появляется кастомное модальное окно для ввода названия заметки
3. После ввода названия и подтверждения:
   - Создаётся новая заметка с `parentId` = ID текущей заметки
   - В текущую заметку вставляется ссылка на созданную заметку
   - Пользователь остаётся в текущей заметке

**Формат ссылки:**
Визуально выделенная ссылка с иконкой: `📄 Название заметки` (кликабельный текст)

**Edge case - несохранённая заметка:**
Если текущая заметка ещё не сохранена (нет ID), система должна автоматически сохранить её перед созданием дочерней заметки.

**Приоритет:**
Обе задачи одинаково важны, реализовать обе.

## Детальный план реализации

### Этап 1: Исправление Enter в слэш-меню

1. В `SlashCommand.ts` добавить `addKeyboardShortcuts()`:
   ```typescript
   addKeyboardShortcuts() {
     return {
       Enter: () => {
         if (this.storage.active) {
           this.options.onSelectCurrent?.();
           return true; // предотвратить стандартное поведение
         }
         return false;
       },
       ArrowDown: () => {
         if (this.storage.active) {
           this.options.onNavigateDown?.();
           return true;
         }
         return false;
       },
       ArrowUp: () => {
         if (this.storage.active) {
           this.options.onNavigateUp?.();
           return true;
         }
         return false;
       },
       Escape: () => {
         if (this.storage.active) {
           this.options.onHide?.();
           return true;
         }
         return false;
       },
     };
   }
   ```

2. Расширить `SlashCommandOptions` для новых callbacks:
   ```typescript
   export interface SlashCommandOptions {
     commands: SlashCommandType[];
     onShow?: (query: string) => void;
     onHide?: () => void;
     onNavigateUp?: () => void;
     onNavigateDown?: () => void;
     onSelectCurrent?: () => void;
   }
   ```

3. В `BlockEditor.vue` передать callbacks при конфигурации extension

4. В `SlashCommandMenu.vue` убрать `document.addEventListener('keydown', ...)` и получать команды через props/events

### Этап 2: Модальное окно для создания вложенной заметки

1. Создать `CreateNestedNoteModal.vue`:
   - Props: `visible`, `parentNoteId`
   - Emits: `create(title: string)`, `close`
   - Стили: центрированное модальное окно, backdrop, input field

2. Стилизация:
   - Фон: белый (тёмная тема: тёмно-серый)
   - Рамка и тень как у SlashCommandMenu
   - Input с placeholder "Название заметки"
   - Кнопки: "Создать" (primary), "Отмена" (secondary)

### Этап 3: Команда /nested

1. Добавить команду в `slashCommands`:
   ```typescript
   {
     id: 'nested',
     title: 'Вложенная заметка',
     description: 'Создать дочернюю заметку',
     icon: '📄',
     keywords: ['nested', 'child', 'subpage', 'вложенная', 'дочерняя'],
     command: (editor) => {
       // Удалить слэш-команду из текста
       editor.chain().focus().deleteRange({...}).run();
       // Показать модальное окно
       showCreateNestedModal();
     },
   }
   ```

2. Обработка создания:
   - Проверить, сохранена ли текущая заметка
   - Если нет - автосохранить
   - Вызвать `notesApi.create({ title, content: '', parentId: currentNoteId })`
   - Вставить ссылку в редактор

### Этап 4: Вставка ссылки на заметку

1. Использовать существующий Link extension или создать кастомный NoteLink
2. Формат вставки:
   ```typescript
   editor.chain()
     .focus()
     .insertContent({
       type: 'text',
       text: `📄 ${noteTitle}`,
       marks: [{ type: 'link', attrs: { href: `/notes/${noteId}` } }]
     })
     .run();
   ```

## Edge Cases и риски

1. **Конфликт горячих клавиш**: Enter в слэш-меню не должен конфликтовать с обычным Enter для новой строки. Решение: проверять `storage.active` перед перехватом.

2. **Race condition при автосохранении**: Если автосохранение долгое, пользователь может закрыть модалку. Решение: показывать loading state.

3. **Потеря фокуса редактора**: После закрытия модалки фокус должен вернуться в редактор. Решение: явно вызывать `editor.commands.focus()`.

4. **Несуществующий parentId**: Если родительская заметка будет удалена, ссылка станет битой. Решение: обрабатывать 404 при клике на ссылку.

5. **Длинное название заметки**: Ссылка может быть очень длинной. Решение: truncate в UI, но хранить полное название.

- Embed/preview содержимого вложенной заметки
- Поиск существующих заметок для создания ссылки (только создание новой)
