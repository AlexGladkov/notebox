# Спецификация: Добавление backdrop к заметке

## Идентификатор
F-07bb86fd-add-backdrop-to-note

## Резюме
Исправить функционал добавления backdrop (обложки) к заметкам. Текущая реализация не позволяет добавить backdrop, потому что кнопка "Добавить обложку" не отображается, когда backdrop отсутствует.

## Проблема
В файле `NoteEditor.vue` компонент `NoteCover` рендерится только при условии `v-if="note.backdropType"`. Это означает, что если у заметки нет backdrop, компонент вообще не отображается, и пользователь не видит кнопку для добавления обложки.

```vue
<!-- Текущая проблемная логика -->
<NoteCover v-if="note.backdropType" ... >
```

## Требования (из интервью)

| Вопрос | Ответ |
|--------|-------|
| Что такое backdrop? | Изображение-фон (как cover/обложка) |
| Отличие от существующей обложки? | Это одно и то же, просто функционал не работает |
| Как показывать кнопку добавления? | При наведении мыши (hover) |
| Какие типы backdrop поддерживать? | Изображения + градиенты (как сейчас в CoverPicker) |
| Позиция иконки относительно backdrop? | Выступает за нижний край (как в Notion) |
| Backdrop автоматический или опциональный? | Опционально - пользователь добавляет по желанию |
| Зависимость иконки и backdrop? | Независимые - любая комбинация возможна |
| Объём работ? | Проверить и исправить весь функционал backdrop |

## Детальный план реализации

### Шаг 1: Исправить NoteEditor.vue
```vue
<!-- БЫЛО -->
<NoteCover v-if="note.backdropType" ... >
  <template #icon>
    <NoteIcon ... />
  </template>
</NoteCover>
<NoteIcon v-if="!note.backdropType" ... />

<!-- СТАНЕТ -->
<NoteCover
  :backdrop-type="note.backdropType"
  :backdrop-value="note.backdropValue"
  :backdrop-position-y="note.backdropPositionY"
  @update="handleCoverUpdate"
>
  <template #icon>
    <div class="icon-over-cover" v-if="note.backdropType">
      <NoteIcon :icon="note.icon" @update="handleIconUpdate" />
    </div>
  </template>
</NoteCover>
<NoteIcon v-if="!note.backdropType" :icon="note.icon" @update="handleIconUpdate" />
```

## Критерии приёмки

1. **Добавление backdrop**: Пользователь может навести мышь на область заголовка заметки и увидеть кнопку "Добавить обложку"
2. **Выбор типа**: После клика открывается CoverPicker с градиентами и изображениями
3. **Отображение**: После выбора backdrop отображается в верхней части заметки
4. **Иконка поверх**: Если есть иконка, она отображается поверх backdrop, частично выступая за нижний край
5. **Изменение**: Пользователь может изменить backdrop на другой
6. **Удаление**: Пользователь может удалить backdrop
7. **Репозиционирование**: Для изображений работает перетаскивание для изменения позиции
8. **Независимость**: Иконка и backdrop независимы - можно добавить только иконку, только backdrop, или оба

## Edge Cases

1. **Заметка без иконки и backdrop**: Оба элемента скрыты, появляются при hover
2. **Только иконка**: Иконка отображается над заголовком, backdrop можно добавить через hover
3. **Только backdrop**: Backdrop отображается, иконку можно добавить отдельно
4. **Невалидный URL изображения**: Graceful degradation - не показывать битое изображение
5. **Очень длинное изображение**: Корректное обрезание и позиционирование

## Риски

1. **Производительность**: Дополнительный рендер NoteCover на каждой заметке - минимальный риск, компонент лёгкий
2. **Визуальные регрессии**: Проверить, что стили не сломались при изменении условий рендеринга

## Вне скоупа

- Загрузка собственных изображений (upload)
- Добавление сплошных цветов как типа backdrop
- Автоматическое добавление backdrop к новым заметкам
- Изменение размера области backdrop
