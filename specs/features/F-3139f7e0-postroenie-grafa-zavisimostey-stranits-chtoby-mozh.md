# Спецификация: Граф зависимостей страниц (Obsidian-style Graph View)

## Краткое описание

Добавить режим визуализации графа связей между страницами, аналогичный Graph View в Obsidian. Пользователь может переключиться в этот режим по отдельной кнопке и видеть все страницы как интерактивный граф с узлами и связями.

## Требования из задачи

1. **Визуализация как в Obsidian** — граф с нодами (страницы) и рёбрами (связи)
2. **Навигация по клику** — при нажатии на ноду происходит переход к соответствующей странице
3. **Отдельный режим** — доступен по отдельной кнопке, работает независимо от редактора
4. **Производительность** — должен работать без тормозов даже при тысячах страниц

## Технический анализ кодовой базы

### Стек технологий
- **Frontend**: Vue 3 + Vite + TypeScript + Tailwind CSS
- **Editor**: TipTap (контент хранится как HTML/JSON)
- **Router**: vue-router (createWebHistory)
- **State**: Composables (useNotes, useTabs, useTheme)

### Существующая структура данных Note
```typescript
interface Note {
  id: string;
  title: string;
  content: string;          // HTML/JSON от TipTap
  parentId?: string | null; // Иерархическая связь
  icon?: string | null;
  // ... другие поля

```

### Внутренние ссылки
Ссылки на страницы в контенте имеют формат:
```html
<a href="/notes/{noteId}">Текст ссылки</a>
```

Парсинг ссылок из HTML-контента:
```typescript
const linkPattern = /href="\/notes\/([a-f0-9-]+)"/g;
```

### Существующий API
- `notesApi.getAll()` — получить все заметки пользователя
- Граф можно построить полностью на клиенте из списка всех заметок

## Детальный план реализации

### Фаза 1: Инфраструктура (1-2 часа)

1. **Утилита парсинга ссылок** (`src/utils/parseNoteLinks.ts`)
   - Функция `extractNoteLinks(content: string): string[]`
   - Возвращает массив ID страниц, на которые есть ссылки

2. **Composable для графа** (`src/composables/useGraph.ts`)
   ```typescript
   interface GraphNode {
     id: string;
     title: string;
     icon?: string;
     x: number;
     y: number;
     vx: number;  // velocity
     vy: number;
   }
   
   interface GraphEdge {
     source: string;
     target: string;
     type: 'parent' | 'link';
   }
   
   interface GraphData {
     nodes: GraphNode[];
     edges: GraphEdge[];
   }
   
   function useGraph(notes: Ref<Note[]>): {
     graphData: ComputedRef<GraphData>;
     // ... методы управления
   }
   ```

### Фаза 2: Алгоритм layout (2-3 часа)

**Force-directed алгоритм** (`src/utils/graphLayout.ts`):
- Силы отталкивания между всеми нодами (Barnes-Hut для O(n log n))
- Силы притяжения по рёбрам (пружины)
- Центростремительная сила к центру viewport
- Затухание для стабилизации

**Оптимизации для 1000+ страниц**:
- Quadtree для ускорения расчёта отталкивания
- Web Worker для вычислений (не блокировать UI)
- Прогрессивный рендеринг (сначала крупные кластеры)
- LOD (Level of Detail): при zoom-out показывать меньше деталей

### Фаза 3: Визуализация (3-4 часа)

**GraphCanvas.vue**:
- Использовать Canvas 2D API (быстрее SVG для большого количества элементов)
- Рендеринг:
  - Рёбра: линии с разным стилем для типов связей
  - Ноды: круги с иконкой/первой буквой названия
  - Labels: названия страниц (скрывать при zoom-out)
- Интерактивность:
  - Pan: перетаскивание canvas
  - Zoom: колёсико мыши + pinch
  - Click: определение ноды под курсором → навигация
  - Hover: подсветка ноды и её связей

**GraphControls.vue**:
- Zoom in/out кнопки
- Reset view (центрировать и сбросить zoom)
- Fullscreen toggle (опционально)

### Фаза 4: Интеграция (1-2 часа)

1. **Маршрут** в `router/index.ts`:
   ```typescript
   {
     path: '/graph',
     name: 'graph',
     component: () => import('../views/GraphView.vue'),
     meta: { requiresAuth: true }
   }
   ```

2. **Кнопка в sidebar** (MainView.vue):
   - Иконка графа рядом с переключателем темы
   - `router.push('/graph')`

3. **Возврат из графа**:
   - Кнопка "Назад к заметкам" → `router.push('/')`
   - При клике на ноду → `router.push('/')` + открыть заметку

## Edge Cases и риски

### Edge Cases

| Случай | Поведение |
|--------|-----------|
| 0 страниц | Показать пустое состояние с подсказкой |
| 1 страница | Показать одну ноду в центре |
| Страница без связей (orphan) | Отображается как изолированная нода |
| Циклические ссылки | Корректно обрабатываются алгоритмом |
| Битые ссылки (на удалённые страницы) | Игнорировать при построении графа |
| Очень длинные названия | Обрезать с многоточием |

### Риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Тормоза при 1000+ страниц | Средняя | Высокое | Web Worker + Barnes-Hut + Canvas |
| Сложность отладки layout | Средняя | Среднее | Логирование, визуализация сил |
| Несовместимость с мобильными | Низкая | Среднее | Touch events для pan/zoom |
