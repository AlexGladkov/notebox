# F-eb6095cd: Добавление CSV импорта/экспорта для таблиц баз данных

## Статус
**Этап**: Спецификация  
**Дата**: 2026-05-10

## Краткое описание

Добавить функциональность импорта и экспорта CSV файлов для существующих таблиц баз данных в заметках. Позволит пользователям загружать структурированные данные из внешних источников и выгружать данные для использования в других приложениях.

## Анализ текущего состояния

### Уже реализовано
Система таблиц баз данных полностью функциональна:

1. **TipTap расширение** (`src/extensions/DatabaseExtension.ts`)
   - Блок базы данных как atom node
   - Ссылка на базу данных по `databaseId`

2. **Компоненты таблицы** (`src/components/BlockEditor/`)
   - `DatabaseBlock.vue` — контейнер с загрузкой данных
   - `DatabaseTable.vue` — отрисовка таблицы
   - `DatabaseCell.vue` — универсальная ячейка
   - Специализированные ячейки: `TextCell`, `NumberCell`, `DateCell`, `BooleanCell`, `SelectCell`, `MultiSelectCell` и др.

3. **Типы колонок** (`src/types/index.ts`)
   - TEXT, NUMBER, BOOLEAN, DATE — основные типы
   - SELECT, MULTI_SELECT — выбор из списка
   - FILE, URL, EMAIL, PHONE — специальные типы
   - FORMULA, RELATION, CREATED_TIME, LAST_EDITED_TIME — вычисляемые

4. **Фильтрация и сортировка** (`src/types/database.ts`)
   - Операторы: equals, contains, isEmpty, isNotEmpty, gt, lt, gte, lte
   - Направления сортировки: asc, desc
   - Представления (views) с сохранением настроек

5. **API** (`src/api/databases.ts`)
   - CRUD для баз данных, колонок, записей
   - Управление представлениями

### Требуется реализовать
- CSV экспорт данных таблицы
- CSV импорт данных в существующую или новую таблицу

## Требования (на основе описания задачи)

> Примечание: Интервью не было проведено из-за недоступности backend. Требования основаны на описании задачи и анализе существующего кода.

### Функциональные требования

#### CSV Экспорт
1. Кнопка экспорта в панели инструментов таблицы (`DatabaseToolbar`)
2. Экспорт всех видимых записей (с учётом фильтров)
3. Формат файла:
   - Первая строка — названия колонок
   - Разделитель — запятая
   - Кодировка — UTF-8 с BOM (для корректного открытия в Excel)
4. Имя файла: `{название_базы}_{дата}.csv`

#### CSV Импорт
1. Кнопка импорта в панели инструментов
2. Два режима:
   - **Добавление** — новые записи добавляются к существующим
   - **Замена** — существующие записи удаляются, загружаются новые
3. Диалог предпросмотра перед импортом:
   - Показ первых 5 строк
   - Сопоставление колонок CSV с колонками таблицы
   - Автоматическое определение типов данных
4. Обработка ошибок:
   - Пропуск строк с некорректными данными
   - Отчёт об ошибках после импорта

### Нефункциональные требования
1. Максимальный размер CSV файла: 5 MB
2. Максимальное количество строк: 10 000
3. Поддержка кириллицы и специальных символов
4. Работа без серверной обработки (парсинг на клиенте)

## Критерии приёмки

### CSV Экспорт
- [ ] Кнопка "Экспорт CSV" видна в панели инструментов
- [ ] При клике скачивается файл с корректным именем
- [ ] Файл корректно открывается в Excel с кириллицей
- [ ] Экспортируются только отфильтрованные записи
- [ ] Все типы данных корректно преобразуются в текст

### CSV Импорт
- [ ] Кнопка "Импорт CSV" видна в панели инструментов
- [ ] Открывается диалог выбора файла
- [ ] Показывается предпросмотр данных
- [ ] Можно сопоставить колонки CSV с колонками таблицы
- [ ] Режим "Добавить" не удаляет существующие записи
- [ ] Режим "Заменить" удаляет существующие записи
- [ ] Показывается отчёт об ошибках (если есть)
- [ ] Типы данных корректно преобразуются при импорте

### Граничные случаи
- [ ] Пустой CSV файл — показать сообщение об ошибке
- [ ] CSV без заголовков — предложить ввести вручную
- [ ] Несовпадение количества колонок — предупреждение
- [ ] Файл больше 5 MB — ошибка с понятным сообщением
- [ ] Некорректный формат данных в ячейках — пропуск с логированием

## Edge Cases и риски

### Риски
1. **Производительность**: Парсинг больших файлов может заблокировать UI
   - Решение: Web Worker для парсинга, прогресс-бар

2. **Потеря данных**: Режим "Заменить" удаляет все записи
   - Решение: Подтверждение с предупреждением

3. **Кодировка**: Файлы в разных кодировках (Windows-1251, UTF-16)
   - Решение: Попытка определить кодировку, fallback на UTF-8

### Edge Cases
- CSV с quoted values содержащими переносы строк
- CSV с escaped quotes внутри значений
- Пустые строки в середине файла
- Дубликаты в заголовках
- Очень длинные значения в ячейках

## Deployment Constraints

Приложение деплоится со следующими ограничениями:
- Subpath routing (не root `/`)
- HTTP (не HTTPS) за nginx reverse proxy
- Все URL относительные, без hardcoded host/port

Для CSV функциональности это означает:
- Download через Blob URL (клиентская генерация)
- Не требуется серверный endpoint для экспорта
- Импорт обрабатывается полностью на клиенте
