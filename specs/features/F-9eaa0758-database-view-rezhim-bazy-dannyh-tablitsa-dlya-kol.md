# Database View: режим базы данных (таблица) для коллекций страниц

## Summary

Улучшение существующей системы CustomDatabase в NoteBox для полноценного табличного представления данных, аналогичного Notion. Фокус на развитии inline-блока базы данных с поддержкой типов колонок: TEXT, SELECT, FILE (медиапревью), DATE, URL.

## Requirements (из интервью)

### Архитектурные решения
| Вопрос | Ответ |
|--------|-------|
| Основа функциональности | Развитие существующей системы CustomDatabase (не создание новой) |
| Место отображения | Inline-блок в редакторе (через /database команду) |
| Типы колонок в первой версии | TEXT, SELECT, FILE (медиапревью), DATE, URL |

### UI/UX решения
| Вопрос | Ответ |
|--------|-------|
| Добавление колонок | Кнопка "+" в конце строки заголовков |
| Управление колонками | Клик на заголовок открывает dropdown с опциями |
| Сортировка и фильтрация | Быстрая сортировка в dropdown колонки, фильтры — отдельной кнопкой над таблицей |
| Колонка FILE | Миниатюры изображений прямо в ячейке |
| Количество миниатюр | Максимум 3, остальные как "+N" |
| Загрузка файлов | Клик на ячейку открывает диалог выбора файлов |
| Создание опций SELECT | На лету при вводе нового значения |
| Добавление строк | Кнопка "+ Новая запись" внизу таблицы |
| Подтверждение удаления | Только для колонок (модальное окно), для записей — без подтверждения |

### Не уточнено (таймаут интервью)
- Превью для не-изображений (PDF, видео) — **рекомендация**: показывать иконку типа файла
- Поддержка dark theme для новых компонентов — **рекомендация**: использовать существующие CSS-переменные

## Детальный план реализации

### Этап 1: Доработка FileCell для медиапревью

```typescript
// Новая структура данных для FILE колонки
interface FileValue {
  files: Array<{
    id: string;
    url: string;
    filename: string;
    contentType: string;
    size: number;
  }>;
}
```

**FileCell.vue**:
1. При клике открывать `<input type="file" multiple accept="image/*">`
2. Загружать файлы через `filesApi.upload()`
3. Сохранять массив файлов в `record.data[columnId]`
4. Отображать до 3 миниатюр (32x32px)
5. Если файлов > 3, показывать бейдж "+N"
6. Для не-изображений показывать иконку типа файла

### Этап 2: Сортировка в dropdown колонки

**DatabaseColumnHeader.vue**:
1. Добавить в dropdown:
   - "Сортировать A→Z" / "Сортировать Z→A" (для текста)
   - "Сортировать по возрастанию" / "по убыванию" (для чисел/дат)
2. Emit события `sort` с параметрами `{ columnId, direction }`

### Этап 3: Кнопка фильтра в тулбаре

**DatabaseToolbar.vue**:
1. Добавить кнопку "Фильтр" рядом с поиском
2. По клику показывать существующий `DatabaseFilter.vue`

## Acceptance Criteria

### FileCell с медиапревью
- [ ] Клик на пустую ячейку FILE открывает диалог выбора файлов
- [ ] Поддержка множественного выбора файлов
- [ ] Миниатюры изображений отображаются в ячейке (32x32px)
- [ ] Максимум 3 миниатюры, остальные как бейдж "+N"
- [ ] Файлы загружаются на сервер через существующий API
- [ ] Не-изображения показывают иконку типа файла

### Сортировка
- [ ] Клик на заголовок колонки открывает dropdown
- [ ] В dropdown есть опции сортировки (Asc/Desc)
- [ ] Выбор сортировки применяется к таблице
- [ ] Активная сортировка визуально отмечена (иконка стрелки)

### Фильтрация
- [ ] Кнопка "Фильтр" в тулбаре над таблицей
- [ ] Клик открывает панель настройки фильтра
- [ ] Фильтр применяется к записям таблицы
- [ ] Активный фильтр визуально отмечен (бейдж или подсветка)

### Общие требования
- [ ] Работает в light и dark темах
- [ ] Подтверждение при удалении колонки
- [ ] Удаление записи без подтверждения (через контекстное меню)
- [ ] Новые опции SELECT создаются на лету при вводе

## Edge Cases и риски

### Edge Cases
1. **Большие файлы** — ограничить размер загружаемого файла (рекомендация: 10MB)
2. **Много файлов в ячейке** — UI корректно показывает "+N"
3. **Невалидные изображения** — показывать fallback иконку
4. **Потеря соединения при загрузке** — показать ошибку, возможность повторить
5. **Удаление колонки с данными** — подтверждение с предупреждением о потере данных

### Риски
1. **Производительность** — много миниатюр могут замедлить рендеринг. Решение: lazy loading, виртуализация при >100 записей
2. **Совместимость бэкенда** — убедиться, что бэкенд поддерживает сохранение массива файлов в data

## Out of Scope

Следующие функции **НЕ** входят в текущую задачу:
- Полноэкранный режим базы данных (отдельная страница)
- Drag-and-drop загрузка файлов
- Превью PDF и видео
- Множественный выбор записей (bulk operations)
- Экспорт данных (CSV, Excel)
- Kanban/Calendar/Gallery views
- Колонки типа FORMULA, RELATION, PERSON
- Изменение ширины колонок перетаскиванием
- Изменение порядка колонок перетаскиванием
