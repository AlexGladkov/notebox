# Спецификация: Визуальное отображение цвета заметки в списке

**ID задачи:** faf268bd  
**Зависимость:** 1d6afa8a (Добавить поле color к модели Note) — выполнена

## Резюме

Добавить визуальное отображение цвета заметки в виде вертикальной цветной полоски слева от элемента заметки в списке (компонент NoteTree). Цвет берётся из поля `color` модели Note. Если `color == null` или пустая строка, полоска не отображается.

## Текущее состояние

- Поле `color: String?` уже добавлено в backend модель `Note` (server/src/main/kotlin/com/notebox/domain/note/Note.kt)
- Поле `color` уже передаётся в `NoteDto` (server/src/main/kotlin/com/notebox/dto/NoteDto.kt)
- Frontend интерфейс `Note` в `src/types/index.ts` **не содержит** поля `color` — требуется добавить
- Компонент `NoteTree.vue` отображает список заметок, но без цветового индикатора

## Требования

### Функциональные требования

1. Слева от каждой заметки в списке отображается вертикальная цветная полоска
2. Цвет полоски соответствует значению поля `color` заметки
3. Если поле `color` равно `null`, `undefined` или пустой строке — полоска не отображается
4. Полоска не влияет на существующую функциональность (выделение, раскрытие, клик)

### Визуальные требования

1. **Ширина полоски:** 3px
2. **Высота полоски:** 100% от высоты элемента заметки
3. **Расположение:** слева от элемента, внутри области .note-item-content
4. **Скругление:** небольшое (border-radius: 2px)
5. **Отступ:** полоска прилегает к левому краю элемента (перед иконкой разворачивания)

### Поддержка тем

- Полоска должна корректно отображаться как в светлой, так и в тёмной теме
- Цвет полоски не меняется в зависимости от темы (используется цвет из поля `color`)

## Критерии приёмки

1. [ ] При наличии значения `color` у заметки, слева отображается цветная полоска указанного цвета
2. [ ] При отсутствии `color` (null/undefined/пустая строка) полоска не отображается
3. [ ] Полоска корректно отображается в светлой теме
4. [ ] Полоска корректно отображается в тёмной теме
5. [ ] Полоска отображается для всех уровней вложенности заметок
6. [ ] Существующая функциональность списка заметок (клик, выделение, раскрытие) работает без изменений
7. [ ] Тип Note во frontend синхронизирован с backend (содержит поле color)

## Граничные случаи

| Случай | Ожидаемое поведение |
|--------|---------------------|
| `color = null` | Полоска не отображается |
| `color = undefined` | Полоска не отображается |
| `color = ""` | Полоска не отображается |
| `color = "#FF0000"` | Красная полоска |
| `color = "rgb(0,255,0)"` | Зелёная полоска |
| `color = "blue"` | Синяя полоска (CSS named color) |
| Невалидный цвет (например "xyz") | Браузер игнорирует, полоска прозрачная |

## Риски и ограничения

1. **Формат цвета:** Предполагается, что значение `color` является валидным CSS-цветом. Валидация формата не производится на уровне UI.
2. **Производительность:** Минимальное влияние — добавляется один простой div-элемент на каждую заметку.
