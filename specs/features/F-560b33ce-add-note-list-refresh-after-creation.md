# Спецификация: Обновление списка заметок после создания

## Идентификатор
F-560b33ce-add-note-list-refresh-after-creation

## Резюме
Обеспечить корректное обновление списка заметок (NoteTree) в боковой панели после создания новой заметки, включая автоматическое раскрытие родительской заметки для отображения только что созданной дочерней заметки.

## Контекст

### Текущая реализация
Анализ кодовой базы показал следующую архитектуру:

1. **Хранение заметок**: `useStorage.ts` - загружает заметки с сервера при монтировании и хранит в реактивной `ref<Note[]>`
2. **Операции с заметками**: `useNotes.ts` - содержит функцию `createNote`, которая:
   - Создает заметку через API (`notesApi.create`)
   - Добавляет возвращенную заметку в массив `notes.value.push(newNote)`
3. **Создание заметок** происходит в двух местах:
   - `MainView.vue`: `handleCreateRootPage()` и `handleCreateSubpage(parentId)` - создание через кнопки в сайдбаре
   - `BlockEditor.vue`: `handleCreateNestedNote()` - создание вложенной заметки из редактора (через slash-команду)
4. **Отображение**: `NoteTree.vue` - рекурсивный компонент, отображающий дерево заметок

### Выявленные проблемы
Текущая реализация имеет несколько потенциальных проблем:

1. При создании вложенной заметки из редактора (`handleNoteCreated` в MainView.vue) заметка добавляется корректно, но механизм обновления может быть ненадежным
2. Отсутствует принудительная перезагрузка списка с сервера после создания (полагается только на локальное добавление)
3. Родительская заметка раскрывается через `expandAllAncestors`, но это происходит не во всех сценариях

## Требования

### Функциональные требования

1. **FR-1**: После создания корневой заметки (без родителя) она должна немедленно появиться в списке заметок в сайдбаре
2. **FR-2**: После создания дочерней заметки:
   - Заметка должна появиться в дереве под родительской
   - Родительская заметка должна автоматически раскрыться (если была свернута)
3. **FR-3**: После создания вложенной заметки из редактора (через slash-команду `/nested`) список должен обновиться и показать новую заметку
4. **FR-4**: Обновление списка должно происходить без полной перезагрузки страницы

### Нефункциональные требования

1. **NFR-1**: Обновление должно быть визуально мгновенным (< 100ms после ответа API)
2. **NFR-2**: Не должно быть дублирования заметок при добавлении в список

## Подход к реализации

### Шаг 1: Аудит текущей реализации
Проверить, что текущая реализация корректно работает во всех сценариях:
- Создание корневой заметки
- Создание дочерней заметки через кнопку в сайдбаре
- Создание вложенной заметки через редактор

### Шаг 2: Исправление найденных проблем
Если обнаружены проблемы:
1. Обеспечить реактивное обновление массива заметок
2. Добавить вызов `expandAllAncestors` там, где он отсутствует
3. При необходимости использовать `Vue.set` или spread-оператор для гарантии реактивности

### Шаг 3: Улучшение надежности
1. Добавить проверку на дубликаты перед добавлением заметки в массив
2. Логировать в консоль для отладки

## Критерии приемки

1. [ ] При нажатии кнопки "+" в сайдбаре создается корневая заметка, которая сразу появляется в списке
2. [ ] При нажатии кнопки "+" на родительской заметке создается дочерняя заметка, родитель раскрывается, дочерняя заметка видна
3. [ ] При создании вложенной заметки через slash-команду в редакторе заметка появляется в дереве
4. [ ] Нет дублирования заметок в списке
5. [ ] Обновление происходит без перезагрузки страницы

## Граничные случаи и риски

### Граничные случаи
1. Создание заметки при медленном соединении - должен отображаться индикатор загрузки
2. Ошибка API при создании - должно выводиться сообщение об ошибке, список не должен изменяться
3. Создание нескольких заметок подряд - все должны появиться в списке

### Риски
1. **Риск**: Потеря реактивности Vue при мутации массива
   - **Митигация**: Использовать методы, сохраняющие реактивность (push, splice, или замена массива)
2. **Риск**: Race condition при быстром создании нескольких заметок
   - **Митигация**: Последовательная обработка или проверка на дубликаты

## Вне области задачи

1. Синхронизация между несколькими вкладками браузера
2. WebSocket/Server-Sent Events для real-time обновлений
3. Оптимистичные обновления (показ заметки до ответа сервера)
4. Drag-and-drop переупорядочивание заметок
5. Анимация появления новой заметки
