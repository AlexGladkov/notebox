# Спецификация: Переключатель тёмной темы

## Идентификатор
F-fb43faf4-add-dark-mode-toggle

## Статус
Реализовано

## Краткое описание
Добавление toggle switch для переключения между светлой и тёмной темой в настройках приложения с сохранением предпочтения пользователя на сервере.

---

## Требования

### Функциональные требования

1. **Toggle Switch для переключения темы**
   - Визуальный переключатель между светлой (слева) и тёмной (справа) темой
   - Иконка солнца для светлой темы, иконка луны для тёмной
   - Плавная анимация переключения (0.2s transition)

2. **Режим системной темы**
   - Чекбокс "Использовать системные настройки"
   - При включённом системном режиме toggle switch становится неактивным (disabled)
   - Автоматическое определение темы по системным настройкам через `prefers-color-scheme`

3. **Три режима работы**
   - `light` — всегда светлая тема
   - `dark` — всегда тёмная тема
   - `system` — следует системным настройкам устройства

4. **Сохранение предпочтений**
   - Локальное сохранение в localStorage (`notebox-theme`)
   - Синхронизация с сервером через API `/api/auth/me` (PATCH)
   - При ошибке сохранения на сервер — отображение сообщения об ошибке

### Нефункциональные требования

1. **Доступность (Accessibility)**
   - Toggle имеет `role="switch"` и `aria-checked`
   - Динамическая `aria-label` описывает действие переключения
   - Фокус-состояние с контрастной обводкой

2. **Отзывчивость**
   - Мгновенное применение темы при переключении
   - Сохранение на сервер происходит асинхронно

## Архитектура

### Frontend

#### Composable: `src/composables/useTheme.ts`
- Реактивное состояние темы (`themeMode`)
- Вычисляемое свойство `effectiveTheme` — реальная применяемая тема
- Методы: `setTheme()`, `cycleTheme()`, `initialize()`
- Слушатель медиа-запроса `prefers-color-scheme`
- Автосохранение в localStorage при изменении

#### Компонент: `src/components/settings/AppearanceSection.vue`
- Toggle switch с иконками солнца/луны
- Чекбокс системной темы
- Интеграция с `authStore.updateProfile()` для сохранения на сервер
- Отображение ошибок сохранения

#### API: `src/api/user.ts`
- `updateProfile({ themePreference })` — PATCH `/api/auth/me`

#### Store: `src/stores/authStore.ts`
- `updateProfile()` — обновление профиля и локального состояния пользователя

#### Типы: `src/services/auth/types.ts`
- `User.themePreference: string`

### Backend (Kotlin/Spring Boot)

#### Контроллер: `AuthController.kt`
- `PATCH /api/auth/me` — обновление профиля пользователя
- Валидация `themePreference` — допустимые значения: `light`, `dark`, `system`

#### Сервис: `UserService.kt`
- `updateUser(id, themePreference)` — обновление пользователя в БД

#### Репозиторий: `UserRepository.kt`
- `update()` — обновление записи в таблице `users`

#### База данных
- Миграция: `V002__add_theme_preference_to_users.sql`
- Добавлено поле `theme_preference VARCHAR(10) DEFAULT 'system'`

## Критерии приёмки

- [x] Toggle switch отображается в секции "Внешний вид" настроек
- [x] Переключение между светлой и тёмной темой работает мгновенно
- [x] Чекбокс "Использовать системные настройки" включает/выключает системный режим
- [x] При включённом системном режиме toggle switch неактивен
- [x] Тема сохраняется в localStorage и восстанавливается при перезагрузке
- [x] Тема синхронизируется с сервером
- [x] Изменение системной темы ОС автоматически применяется при режиме `system`
- [x] Все элементы приложения корректно отображаются в тёмной теме
- [x] Accessibility: toggle имеет корректные ARIA-атрибуты

## Пограничные случаи

1. **localStorage недоступен**
   - Тема всё равно работает, просто не сохраняется локально
   - Выводится предупреждение в консоль

2. **Ошибка сохранения на сервер**
   - Локальная тема применяется немедленно
   - Пользователю показывается сообщение об ошибке
   - При следующем входе тема может отличаться от ожидаемой

3. **Несовместимый браузер без `matchMedia`**
   - Системный режим работает как светлая тема по умолчанию

4. **Демо-пользователь**
   - Тема работает нормально
   - Сохранение на сервер работает для демо-сессии

## Риски и ограничения

1. **CSS-переменные**: все компоненты должны использовать CSS-переменные или классы `.dark` для корректного отображения в тёмной теме
2. **Сторонние компоненты**: библиотечные компоненты могут требовать дополнительной стилизации для тёмной темы
3. **Производительность**: множественные перерисовки при частом переключении минимизированы благодаря CSS-классам на `<html>`
