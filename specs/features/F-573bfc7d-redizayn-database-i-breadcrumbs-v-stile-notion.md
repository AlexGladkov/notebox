# Редизайн /database и breadcrumbs в стиле Notion

## Резюме

Обновление визуального стиля и функциональности компонентов базы данных (Database) и хлебных крошек (Breadcrumbs) для приближения к дизайну Notion. Включает добавление табов-вьюх, тулбара с фильтрами/сортировкой/поиском, цветных тегов и улучшенного визуала.

## Собранные требования

### Общее
- **Приоритет**: Полная реализация всех элементов со скриншота
- **Тёмная тема**: Адаптировать только новые элементы (существующие стили не трогать)

### Database

#### Табы-вьюхи
- **Функционал**: Сохраняемые вьюхи с настройками фильтров, сортировок и видимых колонок
- **Хранение**: На backend (в структуре базы данных через API)
- **UI**: Горизонтальные табы над таблицей с возможностью создания/переключения/удаления

#### Тулбар
- **Filter**: Базовый — один фильтр по одной колонке (выбор колонки, оператор, значение)
- **Sort**: Базовый — сортировка по одной колонке (asc/desc)
- **Search**: Поиск по всем колонкам таблицы (фильтрация строк в реальном времени)
- **Share**: Копирует URL текущей страницы в буфер обмена

#### Теги (Select/MultiSelect)
- Пользователь создаёт кастомные теги
- Цвет назначается автоматически из предустановленной палитры
- Палитра: зелёный, жёлтый, розовый, фиолетовый, синий, оранжевый, красный, серый

#### Чекбоксы строк
- Визуально присутствуют (как в Notion)
- Функционал массовых операций — вне скоупа (только UI)

#### Иконки записей
- Фиксированная иконка документа для всех строк в колонке Name
- Кастомные иконки для записей — вне скоупа

#### Кнопка "New"
- Остаётся под таблицей (как сейчас реализовано)
- Кнопка в тулбаре — вне скоупа

#### Типы колонок
- Все существующие типы сохраняются: Text, Number, Select, MultiSelect, Date, Boolean, Email, URL, Phone, File, Relation, Formula, CreatedTime, LastEditedTime, Person
- Status и Access — стандартные Select-колонки с цветными опциями

### Breadcrumbs

- **Изменения**: Только визуальный рестайлинг под Notion
- **Логика**: Остаётся без изменений (навигация по иерархии заметок)
- **Стиль**: Компактные элементы, иконки страниц, разделители-стрелки
- **Дополнительные кнопки справа**: Вне скоупа
- **Dropdown-меню**: Вне скоупа

## Детальный план реализации

### Этап 1: Типы и структуры данных

1. Создать `src/types/database.ts`:
```typescript
interface DatabaseView {
  id: string;
  name: string;
  filter?: DatabaseFilter;
  sort?: DatabaseSort;
  visibleColumns?: string[];
}

interface DatabaseFilter {
  columnId: string;
  operator: 'equals' | 'contains' | 'gt' | 'lt' | 'isEmpty' | 'isNotEmpty';
  value: any;
}

interface DatabaseSort {
  columnId: string;
  direction: 'asc' | 'desc';
}
```

2. Расширить тип `Database` в `src/types/index.ts`:
```typescript
interface Database {
  // ... existing fields
  views: DatabaseView[];
}
```

### Этап 2: API для вьюх

1. Добавить в `src/api/databases.ts`:
   - `createView(databaseId, view)` — создание вьюхи
   - `updateView(databaseId, viewId, view)` — обновление
   - `deleteView(databaseId, viewId)` — удаление

2. Обновить `src/composables/useDatabases.ts`:
   - Методы для CRUD операций с вьюхами
   - Реактивное состояние текущей вьюхи

### Этап 3: Компоненты Database

1. **DatabaseViewTabs.vue**:
   - Горизонтальный список табов
   - Кнопка "+" для создания новой вьюхи
   - Контекстное меню (переименовать, удалить)
   - Emit: `select-view`, `create-view`, `delete-view`

2. **DatabaseToolbar.vue**:
   - Контейнер для Filter, Sort, Search, Share
   - Отображение активных фильтров/сортировок как бейджей

3. **DatabaseFilter.vue**:
   - Dropdown с выбором колонки
   - Выбор оператора (зависит от типа колонки)
   - Ввод значения
   - Кнопка очистки

4. **DatabaseSort.vue**:
   - Dropdown с выбором колонки
   - Переключатель asc/desc
   - Кнопка очистки

5. **DatabaseSearch.vue**:
   - Иконка поиска, при клике — расширяемое поле ввода
   - Debounce 300ms

6. **Share функционал**:
   - Кнопка копирует `window.location.href` в clipboard
   - Toast-уведомление "Ссылка скопирована"

### Этап 4: Обновление DatabaseTable

1. Добавить колонку с чекбоксами слева:
   - Чекбокс в заголовке (select all) — визуально, без логики
   - Чекбоксы у строк — визуально, без логики

2. Добавить иконку документа перед текстом в колонке Name:
   - SVG-иконка страницы (как в Notion)

3. Применение фильтров и сортировки:
   - Props: `filter`, `sort`, `searchQuery`
   - Computed `filteredRecords` с применением всех условий

### Этап 5: Цветные теги

1. Обновить `SelectCell.vue` и `MultiSelectCell.vue`:
   - Палитра из 8 цветов
   - При создании нового тега — автоматически назначается следующий цвет из палитры
   - Цвет сохраняется в `SelectOption.color`

2. Стили тегов:
   - Скруглённые бейджи с цветным фоном и текстом
   - Адаптация для тёмной темы (приглушённые цвета)

### Этап 6: Breadcrumbs рестайлинг

1. Обновить `src/components/Breadcrumbs.vue`:
   - Убрать фоновый цвет и border
   - Уменьшить отступы (более компактно)
   - Обновить стиль разделителей (тонкие стрелки)
   - Hover-эффект как в Notion (подчёркивание или фон)
   - Иконки заметок перед названиями

## Acceptance Criteria

### Database

- [ ] Табы-вьюхи отображаются над таблицей

- [ ] Можно переключаться между вьюхами

- [ ] Сортировка: выбор колонки, направления — строки сортируются

- [ ] Share: клик копирует URL, показывается toast

- [ ] Все элементы корректно отображаются в тёмной теме

- [ ] Навигация по клику работает как прежде
- [ ] Корректное отображение в тёмной теме

## Edge Cases и риски

### Edge Cases

1. **Пустая база данных**: Табы и тулбар должны отображаться даже без записей
2. **Длинные названия вьюх**: Обрезка с ellipsis, tooltip с полным названием
3. **Много вьюх**: Горизонтальный скролл или overflow-меню
4. **Фильтр без результатов**: Сообщение "Нет записей, соответствующих фильтру"
5. **Поиск по пустым ячейкам**: Пустые значения не должны матчиться
6. **Удаление последней вьюхи**: Запретить, показать предупреждение

### Риски

1. **Backend API**: Требуется доработка API для хранения вьюх — может потребоваться координация с backend-разработчиком
2. **Производительность**: Фильтрация/поиск больших таблиц — использовать debounce и виртуализацию при необходимости
3. **Миграция данных**: Существующие базы данных не имеют вьюх — создавать дефолтную вьюху "All" при загрузке

## Вне скоупа (Out of Scope)

- Кнопки в правой части breadcrumbs (Share, профиль)
- Dropdown-меню в breadcrumbs для навигации к соседним заметкам
- Массовые операции с чекбоксами (удаление, редактирование нескольких строк)
- Кастомные иконки для записей
- Кнопка "New" в тулбаре
- Продвинутые фильтры (AND/OR, несколько условий)
- Многоуровневая сортировка
- Группировка записей
- Выбор цвета тега пользователем
- Скрытие/показ колонок через UI

## Зависимости

- Backend API должен поддерживать хранение `views` в объекте Database
- Если API не готов, вьюхи временно можно хранить в localStorage

## Оценка сложности

- **Database**: Высокая (новые компоненты, интеграция, API)
- **Breadcrumbs**: Низкая (только CSS изменения)
- **Общее время**: ~3-4 дня разработки
