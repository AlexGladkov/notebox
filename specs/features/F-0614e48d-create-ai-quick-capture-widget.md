# F-0614e48d: AI Quick Capture Widget

## Краткое описание

Создание плавающей кнопки (FAB - Floating Action Button) для быстрого захвата информации с поддержкой голосового ввода, OCR с фотографий и быстрого текста. AI автоматически организует захваченные элементы в подходящие заметки с использованием семантического сопоставления.

## Статус интервью

**Примечание:** Полноценное интервью с пользователем не было проведено из-за недоступности бэкенда. Спецификация основана на анализе кодовой базы и описании задачи. Ключевые решения помечены как допущения (🔶) и требуют подтверждения перед реализацией.

## Требования

### Функциональные требования

#### 1. Плавающая кнопка (FAB)
- Фиксированная позиция в правом нижнем углу экрана
- Сворачивается до компактной иконки "+"
- При нажатии раскрывается меню с тремя типами захвата
- Анимация раскрытия/сворачивания
- Поддержка тёмной и светлой темы

#### 2. Типы захвата

##### 2.1 Быстрый текст
- Модальное окно с textarea для ввода текста
- Placeholder: "Быстрая заметка..."
- Кнопки: Отмена / Сохранить
- 🔶 **Допущение**: Реализуется первым как MVP

##### 2.2 Голосовое мемо
- Использование Web Speech API для распознавания речи
- Индикатор записи (пульсирующая анимация)
- Отображение распознанного текста в реальном времени
- Кнопки: Остановить запись / Отмена / Сохранить
- 🔶 **Допущение**: Реализуется во второй фазе

##### 2.3 OCR с фотографии
- Загрузка изображения через input type="file" или drag-and-drop
- Поддержка камеры на мобильных устройствах (capture="environment")
- Предпросмотр изображения
- Вызов AI API для OCR
- Редактирование распознанного текста перед сохранением
- 🔶 **Допущение**: Реализуется во второй фазе

#### 3. AI авто-организация

##### 3.1 Inbox (Входящие)
- Создание специальной заметки "📥 Inbox" для временного хранения
- Все захваченные элементы сначала попадают во Inbox
- 🔶 **Допущение**: Inbox создаётся автоматически при первом использовании

##### 3.2 Семантическое сопоставление
- AI анализирует содержимое захваченного элемента
- Сравнивает с существующими заметками по семантическому сходству
- Предлагает 1-3 наиболее подходящие заметки для размещения
- Пользователь может:
  - Выбрать предложенную заметку
  - Оставить в Inbox
  - Создать новую заметку
- 🔶 **Допущение**: Семантический матчинг реализуется через новый API endpoint

##### 3.3 Интеграция с существующим AI
- Использование существующего `aiApi` из `src/api/ai.ts`
- Добавление новых методов:
  - `aiApi.ocr(imageBase64: string): Promise<string>`
  - `aiApi.findRelatedNotes(text: string, noteIds: string[]): Promise<RelatedNote[]>`

### Нефункциональные требования

- Виджет должен работать на всех страницах приложения (MainView, GraphView)
- Отзывчивый дизайн для мобильных устройств
- Поддержка offline режима (сохранение в локальное хранилище с последующей синхронизацией)
- Accessibility: поддержка клавиатурной навигации и screen readers

## Детальный план реализации

### Фаза 1: MVP (только текстовый захват)

1. **Создать базовый FAB компонент** (`QuickCaptureButton.vue`)
   - Круглая кнопка с иконкой "+"
   - Позиция: `fixed bottom-6 right-6`
   - Анимация при hover/focus

2. **Создать модал текстового ввода** (`TextCaptureModal.vue`)
   - Textarea с autofocus
   - Кнопки управления
   - Валидация (минимум 1 символ)

3. **Создать composable** (`useQuickCapture.ts`)
   - Функция `captureText(text: string)`
   - Создание/получение Inbox заметки
   - Добавление текста в Inbox

4. **Интеграция с MainView.vue**
   - Добавить компонент FAB
   - Подключить обработчики

### Фаза 2: Голосовой ввод и OCR

5. **Создать composable для Speech Recognition** (`useSpeechRecognition.ts`)
   - Проверка поддержки браузером
   - Управление записью
   - Обработка результатов

6. **Создать модал голосового ввода** (`VoiceCaptureModal.vue`)
   - Индикатор записи
   - Отображение текста
   - Управление записью

7. **Добавить OCR API** (`src/api/ai.ts`)
   - Новый метод `ocr()`
   - Mock реализация для dev режима

8. **Создать модал для OCR** (`PhotoCaptureModal.vue`)
   - Загрузка изображения
   - Предпросмотр
   - Отображение распознанного текста

### Фаза 3: AI авто-организация

9. **Добавить семантический поиск** (`src/api/ai.ts`)
   - Метод `findRelatedNotes()`
   - Mock реализация

10. **Создать модал предложений** (`NoteSuggestionModal.vue`)
    - Список предложенных заметок
    - Опция "Оставить в Inbox"

11. **Интеграция в flow захвата**

## Edge Cases и обработка ошибок

### Текстовый захват
- Пустой ввод → показать ошибку валидации
- Offline режим → сохранить локально, синхронизировать позже
- Inbox удалён пользователем → пересоздать автоматически

### Голосовой ввод
- Браузер не поддерживает Speech API → показать информативное сообщение
- Пользователь не дал доступ к микрофону → показать инструкции
- Ошибка распознавания → предложить ввести текст вручную

### OCR
- Неподдерживаемый формат изображения → показать ошибку
- Слишком большой файл → показать ограничение (max 10MB)
- AI API недоступен → показать mock результат / ошибку

### AI организация
- Нет похожих заметок → предложить создать новую или оставить в Inbox
- AI API недоступен → сохранить в Inbox без предложений

## Out of Scope (вне рамок данной задачи)

- Распознавание голоса на других языках кроме русского
- Редактирование существующих заметок через виджет
- Bulk-захват нескольких элементов одновременно
- Интеграция с внешними сервисами (Google Keep, Notion и т.д.)
- История захваченных элементов
- Теги для захваченных элементов
- Расписание автоматической организации Inbox

## Технические риски и митигации

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Web Speech API не работает во всех браузерах | Высокая | Средняя | Graceful degradation, показ сообщения о неподдержке |
| OCR API требует бэкенд | Высокая | Высокая | Mock реализация для dev, настраиваемый endpoint |
| Семантический поиск требует ML модель | Высокая | Высокая | Начать с простого текстового поиска, улучшать итеративно |
| Конфликты с offline режимом | Средняя | Средняя | Использовать существующий `offlineStore` |

## Примерная структура компонента FAB

```vue
<template>
  <div class="fixed bottom-6 right-6 z-50">
    <!-- Кнопки меню (появляются при раскрытии) -->
    <transition name="fade-up">
      <div v-if="isOpen" class="flex flex-col gap-3 mb-3">
        <button @click="openVoiceCapture" class="fab-item" title="Голосовая заметка">
          🎤
        </button>
        <button @click="openPhotoCapture" class="fab-item" title="Фото OCR">
          📷
        </button>
        <button @click="openTextCapture" class="fab-item" title="Текстовая заметка">
          ✏️
        </button>
      </div>
    </transition>
    
    <!-- Главная кнопка -->
    <button
      @click="toggleMenu"
      :class="['fab-main', { 'rotate-45': isOpen }]"
      aria-label="Быстрый захват"
    >
      ➕
    </button>
  </div>
</template>
```
