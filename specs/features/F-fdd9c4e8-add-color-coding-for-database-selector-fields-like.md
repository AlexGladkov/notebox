# F-fdd9c4e8: Цветовая кодировка для selector-полей в стиле Notion

## Резюме

Реализовать полноценную цветовую кодировку для полей типа SELECT и MULTI_SELECT в базах данных. Каждая опция должна иметь настраиваемый цвет фона из предопределённой палитры. Цвета должны отображаться единообразно во всех представлениях: ячейках таблицы, карточках Kanban и dropdown-ах фильтров.

## Анализ текущего состояния

### Что уже реализовано

1. **Палитра цветов** (`src/types/database.ts`):
   - `TAG_COLOR_PALETTE` содержит 8 цветов: green, yellow, pink, purple, blue, orange, red, gray
   - Каждый цвет имеет варианты для светлой и тёмной темы

2. **SelectCell.vue** и **MultiSelectCell.vue**:
   - Опции отображаются с цветным фоном
   - Функция `getOptionStyle()` возвращает стили на основе цвета
   - При создании новой опции цвет назначается автоматически (цикл по палитре)
   - Поддержка тёмной темы

3. **Схема данных** (`server/src/main/kotlin/.../ColumnsTable.kt`):
   - Поле `options` хранит JSON-строку с опциями
   - Опции уже содержат поле `color`

### Выявленные пробелы

1. **KanbanCard.vue** (строки 117-134):
   - SELECT/MULTI_SELECT поля отображаются как текст: `displayValue = option?.label || ''`
   - Цвета не визуализируются

2. **DatabaseFilter.vue** (строки 59-72):
   - Используется стандартный `<select>` для выбора значений
   - Нет цветовой визуализации опций

3. **Отсутствует UI для изменения цвета**:
   - Пользователь не может выбрать другой цвет для существующей опции
   - Нет color picker в контекстном меню опции

## Требования к реализации

### 1. UI для изменения цвета опции

**Где**: SelectCell.vue, MultiSelectCell.vue

**Поведение**:
- При наведении на опцию в dropdown показывать кнопку настроек (иконка "...")
- Клик по кнопке открывает подменю с палитрой цветов
- Палитра: 8 цветов из `TAG_COLOR_PALETTE` в виде цветных кружков
- Клик по цвету обновляет опцию и закрывает подменю
- Изменение сохраняется на сервер через существующий API

### 2. Цвета в карточках Kanban

**Где**: KanbanCard.vue

**Поведение**:
- SELECT поля отображаются как цветные теги (аналогично ячейкам таблицы)
- MULTI_SELECT поля отображаются как несколько цветных тегов
- Использовать те же стили что в SelectCell (`getOptionStyle`)

### 3. Цвета в фильтрах

**Где**: DatabaseFilter.vue

**Поведение**:
- Заменить стандартный `<select>` на custom dropdown с цветными опциями
- Каждая опция отображается как цветной тег
- Поиск по опциям (опционально)

## Детальный план реализации

### Этап 1: Компонент ColorPicker

```vue
<!-- ColorPicker.vue -->
<template>
  <div class="color-picker">
    <div
      v-for="color in TAG_COLOR_PALETTE"
      :key="color.name"
      class="color-swatch"
      :class="{ selected: selectedColor === color.name }"
      :style="{ backgroundColor: isDark ? color.dark : color.light }"
      @click="$emit('select', color.name)"
    />
  </div>
</template>
```

2. По клику показать ColorPicker рядом с опцией
3. Emit события `updateOptionColor` с `{ optionId, color }`
4. В родительском компоненте обновить опцию и сохранить

```typescript
// В displayFields computed:

    displayValue = option?.label || '';

  }
  break;
```

## Критерии приёмки

1. [ ] Пользователь может изменить цвет любой опции через контекстное меню
2. [ ] Изменение цвета сохраняется на сервер и не теряется при перезагрузке
3. [ ] В карточках Kanban SELECT/MULTI_SELECT отображаются как цветные теги
4. [ ] В фильтрах опции отображаются с цветами
5. [ ] Все цвета корректно отображаются в тёмной теме
6. [ ] Цветовая кодировка единообразна во всех местах (таблица, kanban, фильтры)

## Граничные случаи и риски

### Граничные случаи

1. **Опция без цвета**: использовать серый цвет по умолчанию
2. **Неизвестный цвет**: если опция имеет цвет, отсутствующий в палитре, использовать серый
3. **Много опций в фильтре**: dropdown должен скроллиться
4. **Длинные названия опций**: текст обрезается с ellipsis

### Риски

1. **Производительность Kanban**: много карточек с множеством тегов могут замедлить рендеринг
   - Митигация: использовать `v-memo` для карточек

2. **Совместимость с существующими данными**: старые опции могут не иметь поле `color`
   - Митигация: fallback на 'gray' или автоназначение при первом рендере

3. **Z-index конфликты**: color picker поверх других dropdown-ов
   - Митигация: использовать Portal или достаточно высокий z-index
