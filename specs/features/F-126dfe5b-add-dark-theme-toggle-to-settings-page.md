# Спецификация: Добавление toggle switch для переключения тёмной темы

## Идентификатор
F-126dfe5b

## Резюме
Добавить переключатель (toggle switch) в настройках приложения для быстрого переключения между светлой и тёмной темой. Сохранять предпочтение пользователя в localStorage.

## Текущее состояние
Анализ кодовой базы показал, что функциональность переключения темы **уже реализована**:

1. **`src/composables/useTheme.ts`** — composable для управления темой:
   - Поддерживает три режима: `'light'`, `'dark'`, `'system'`
   - Сохраняет выбор в localStorage (ключ: `notebox-theme`)
   - Отслеживает системные настройки через `prefers-color-scheme`
   - Применяет тему через добавление/удаление класса `dark` на `<html>`

2. **`src/components/settings/AppearanceSection.vue`** — секция настроек внешнего вида:
   - Отображает три карточки для выбора темы (светлая, тёмная, системная)
   - Применяет тему мгновенно при клике
   - Синхронизирует выбор с сервером через `authStore.updateProfile()`
   - Имеет стили для тёмной темы

3. **`src/components/settings/SettingsModal.vue`** — модальное окно настроек:
   - Содержит боковую панель с секциями (профиль, внешний вид, уведомления)
   - Секция "Внешний вид" (`AppearanceSection`) уже включена

## Требования из описания задачи
- Добавить **toggle switch** (переключатель) для темы
- Переключение между светлой и тёмной темой
- Сохранение предпочтения в localStorage

## Интерпретация задачи
Поскольку функциональность переключения темы уже существует, но использует UI в виде карточек, задача интерпретируется как:

**Вариант A (рекомендуемый):** Заменить текущие карточки на компактный toggle switch с сохранением опции "системная тема"

**Вариант B:** Добавить простой toggle switch только для light/dark без системной опции

Выбран **Вариант A** — это сохраняет всю текущую функциональность и улучшает UX более компактным интерфейсом.

## Детальный план реализации

### 1. Изменение `AppearanceSection.vue`

#### Новый UI компонент:
```
┌─────────────────────────────────────────────────┐
│  Внешний вид                                    │
├─────────────────────────────────────────────────┤
│                                                 │
│  Тема                                           │
│  ┌─────────────────────────────────────────┐   │
│  │ ☀️ Светлая    [====○    ]    🌙 Тёмная  │   │
│  └─────────────────────────────────────────┘   │
│                                                 │
│  ☐ Использовать системные настройки            │
│    (автоматически светлая/тёмная по системе)   │
│                                                 │
└─────────────────────────────────────────────────┘
```

#### Логика работы:
1. Toggle switch переключает между `'light'` и `'dark'`
2. Чекбокс "Использовать системные настройки" устанавливает режим `'system'`
3. Когда чекбокс активен, toggle switch показывает текущую эффективную тему (но disabled)
4. При снятии чекбокса — применяется последняя выбранная тема (light или dark)

#### Компоненты toggle switch:
- Контейнер с иконками солнца и луны по бокам
- Переключатель-слайдер по центру
- Плавная анимация перехода (transition: 0.2s)
- Доступность: `role="switch"`, `aria-checked`, `aria-label`

## Критерии приёмки

### Функциональные
- [ ] Toggle switch переключает тему между светлой и тёмной
- [ ] Изменение темы применяется мгновенно (без перезагрузки)
- [ ] Выбор сохраняется в localStorage (ключ `notebox-theme`)
- [ ] Чекбокс "Системные настройки" активирует режим `'system'`
- [ ] При активном системном режиме тема следует за настройками ОС
- [ ] При перезагрузке страницы тема восстанавливается из localStorage

### UI/UX
- [ ] Toggle switch имеет плавную анимацию (200ms)
- [ ] Иконки солнца и луны отображаются по бокам переключателя
- [ ] Текущая тема визуально выделена
- [ ] Стили корректно отображаются в тёмной теме

### Доступность (a11y)
- [ ] Toggle switch имеет `role="switch"` и `aria-checked`
- [ ] Чекбокс имеет связанный `<label>`
- [ ] Переключение возможно с клавиатуры (Space/Enter)
- [ ] Focus state виден

### Совместимость
- [ ] Работает в современных браузерах (Chrome, Firefox, Safari, Edge)
- [ ] Корректно отображается на мобильных устройствах
- [ ] Fallback при недоступности localStorage (консольное предупреждение, но без ошибок)

## Граничные случаи

1. **localStorage недоступен** — тема не сохраняется, используется системная; предупреждение в консоли
2. **prefers-color-scheme не поддерживается** — системный режим всегда возвращает светлую тему
3. **Изменение системной темы в реальном времени** — приложение должно реагировать мгновенно (в режиме 'system')
4. **Некорректное значение в localStorage** — игнорируется, используется 'system'

## Риски и ограничения
