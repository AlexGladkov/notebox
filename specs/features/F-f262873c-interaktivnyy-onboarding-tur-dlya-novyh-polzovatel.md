# Спецификация: Интерактивный onboarding-тур для новых пользователей

**ID:** F-f262873c  
**Статус:** Draft  
**Дата:** 2026-05-27

## Краткое описание

Реализация интерактивного пошагового тура по ключевым функциям приложения NoteBox для новых пользователей. Тур автоматически запускается при первом входе после регистрации, подсвечивает элементы интерфейса с пояснениями и может быть запущен повторно из настроек.

## Требования

### Функциональные требования

1. **Интеграция библиотеки для туров**
   - Использовать Driver.js (v1.x) — легковесная библиотека (~5KB), без зависимостей
   - Причины выбора: современный API, TypeScript support, хорошо работает с Vue 3, активно поддерживается
   - Альтернатива Shepherd.js отклонена из-за большого размера (~90KB) и зависимости от Popper.js

2. **Шаги тура (7 шагов)**
   - **Шаг 1: Приветствие** — оверлей без привязки к элементу, знакомство с NoteBox
   - **Шаг 2: Редактор заметок** — подсветка области редактора (NoteEditor), объяснение базового редактирования
   - **Шаг 3: Slash-команды** — подсветка редактора с инструкцией набрать `/` для вызова команд
   - **Шаг 4: Wiki-ссылки** — объяснение синтаксиса `[[название]]` для связывания заметок
   - **Шаг 5: Граф связей** — подсветка кнопки "Граф связей" в сайдбаре
   - **Шаг 6: Быстрый захват** — подсветка FAB-кнопки (QuickCaptureButton) в правом нижнем углу
   - **Шаг 7: Шаблоны** — завершающий шаг с предложением создать первую заметку из шаблона

3. **Условный запуск**
   - Тур запускается автоматически при первом входе после регистрации
   - Флаг `onboardingCompleted` хранится в localStorage (ключ: `notebox_onboarding_completed`)
   - Дополнительно можно хранить на сервере в профиле пользователя (опционально, для синхронизации между устройствами)

4. **Управление туром**
   - Кнопка "Пропустить" на каждом шаге
   - Кнопки "Назад" / "Далее" для навигации
   - Закрытие по клику на overlay или нажатию Escape
   - При пропуске тур считается завершённым (не показывать повторно автоматически)

5. **Шаблоны быстрого старта (3 новых шаблона)**
   - **Персональный журнал** — для ежедневных записей, рефлексии, благодарностей
   - **База знаний** — структурированный шаблон для хранения информации по темам
   - **Задачи** — простой трекер задач с чеклистами и приоритетами
   - Категория шаблонов: `quickstart` (отдельная от существующих work/personal/learning)
   - Шаблоны отображаются на последнем шаге тура и в галерее шаблонов

6. **Кнопка "Пройти тур заново" в настройках**
   - Добавить новую секцию "Справка" (Help) в SettingsSidebar
   - Секция содержит кнопку "Пройти тур заново"
   - При клике сбрасывается флаг и запускается тур

### Нефункциональные требования

1. **Производительность**
   - Библиотека Driver.js загружается lazy (dynamic import) только при запуске тура
   - Не влияет на время первоначальной загрузки приложения

2. **Доступность (a11y)**
   - Фокус перемещается на активный элемент шага
   - Поддержка навигации с клавиатуры (Tab, Enter, Escape)
   - ARIA-атрибуты для элементов тура

3. **Responsive**
   - Тур корректно работает на мобильных устройствах
   - Позиция tooltip адаптируется к размеру экрана

4. **Тёмная тема**
   - Стили тура адаптируются под текущую тему (light/dark)

## Edge Cases и риски

### Edge Cases

1. **Пользователь закрывает вкладку во время тура**
   - Тур не помечается как завершённый, при следующем входе начнётся заново

2. **Элемент шага не найден на странице**
   - Driver.js автоматически пропускает шаг, тур продолжается

3. **Пользователь уже создал заметки до тура**
   - Тур работает одинаково вне зависимости от наличия заметок

4. **Очистка localStorage**
   - Тур запустится заново при следующем входе

5. **Демо-пользователь**
   - Для демо-пользователей тур также запускается (отдельный localStorage)

### Риски

1. **Изменение структуры DOM**
   - Риск: селекторы элементов могут перестать работать
   - Митигация: использовать data-атрибуты (`data-onboarding="..."`)

2. **Конфликт с другими модальными окнами**
   - Риск: модальное окно может перекрыть тур
   - Митигация: z-index тура выше всех модалов, тур не запускается при открытом модале

3. **Локализация**
   - Текущая реализация только на русском языке
   - Будущее расширение: i18n для текстов тура
